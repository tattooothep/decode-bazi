# zone5-map — โยงคัมภีร์ PD → ส่วนไหนของเข็ม D

> เข็ม D = จุดกึ่งกลาง (midpoint) + หน้าปัด 90° + ภาพดาว A+B−C · **เรขาคณิตล้วน** (ไม่ต้องมีลิขสิทธิ์)
> Zone 5 = คัมภีร์รากคณิต PD สะอาด เพื่ออ้างที่มา/เขียน ⓘ อธิบายให้ user (ไม่ใช่ลอกความหมายภาพดาว Witte)

## ลำดับความเป็นบรรพบุรุษ (timeline)

```
Ptolemy (ค.ศ. 2) ── หารวงกลม 2/3/4/6 = aspect · antiscia (beholding/equal power) · Part of Fortune (A+B−C)
        │
Kepler 1619 ─────── ให้เหตุผล harmonic + เพิ่ม aspect จากรูปหลายเหลี่ยม (5,8-fold → 72°,45°,135°)
        │
Witte/Hamburg 1920s ─ จุดกึ่งกลาง (Halbsumme) + หน้าปัด 90° + ภาพดาว A+B−C  ← เข็ม D อยู่ตรงนี้
```

เข็ม D สืบทอด: **มุม/harmonic (Ptolemy+Kepler) + จุดกระจก (Ptolemy) + เลขคณิต A+B−C (Ptolemy)** → รวมเป็นเทคนิค midpoint-on-dial ที่ Witte ประกอบขึ้น

## ประเมินสรุป: ใช้เป็น "คัมภีร์รากคณิต" ให้เข็ม D ได้ไหม?

**ได้ — ในฐานะรากเรขาคณิต/ที่มา 3 ใน 4 กลไก แบบ verbatim สะอาด · แต่ไม่ทดแทนคัมภีร์ความหมาย Witte**

- ✅ 3 กลไก (หน้าปัด-มุมฉาก, antiscia, A+B−C) มี **verbatim จาก Ptolemy 1822 (PD ทั่วโลก)** ครบ — เขียน ⓘ / บทความ /article อธิบายที่มาเข็ม D ได้ทันที โดยไม่ต้องแต่งเอง (ตรงกฎ canon = source of truth)
- 🟡 กลไก harmonic ขั้นสูง (45°/135°) มี **Kepler 1619 (PD)** รองรับระดับบรรณานุกรม/แนวคิด (verbatim ละตินยังติดปัญหา OCR)
- ❌ **ไม่มีคัมภีร์คลาสสิกที่พูดคำว่า "midpoint/จุดกึ่งกลาง" ตรง ๆ** — เทคนิคนี้เป็นของ Witte จริง (ยังหา verbatim ไม่ได้) แต่ **คณิต (A+B)/2 เป็นเรขาคณิตล้วน ใช้ได้เสรี** อยู่แล้ว → เข็ม D ไม่ติดลิขสิทธิ์
- ❌ Zone 5 **ให้ความหมายภาพดาว (Sun/Moon = ..., Mars/Saturn = ...) ไม่ได้** — ความหมายเป็นของ Witte ต้องรอ verbatim ตาม `00-source-policy.md`

**บรรทัดล่างสุด:** Zone 5 ทำให้เข็ม D **มีที่มาถูกลิขสิทธิ์ที่อ้างได้จริง** (ราก Ptolemy + Kepler PD) สำหรับส่วน "ทำไมเลือก 90°/harmonic/mirror/สมการผลรวม" — เพียงพอสำหรับ ⓘ อธิบาย + บทความเรขาคณิต · แต่ **ยังต้องหาคัมภีร์ Witte verbatim แยกต่างหาก** สำหรับ "ความหมาย" ของภาพดาว
