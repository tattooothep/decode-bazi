# r2j-1 · ตรวจ "จุดที่ AI ยูเรเนียนถูกสั่งให้บอกขาด" (READ-ONLY audit)

วันที่ 4 ก.ค. 2026 · ตรวจ 4 ไฟล์: `engine.ts` · `packet.ts` · `render.ts` · `auslosung.ts`
โจทย์: ไล่ทุกจุดที่ prompt/packet สั่ง AI ให้ "บอกขาด / ยังไม่ / notAvailable / ห้าม…เพราะไม่มี / degrade / uncertain"
แล้วจัดเป็น 3 กลุ่ม: (ก) หายเมื่อ TNP r391 ลง · (ข) หายไม่ได้ถ้า user ไม่ให้เวลาเกิด · (ค) ยังขาดจริง = ต้อง implement เพิ่ม

> **ยืนยันความซื่อสัตย์:** ทุกจุด "ขาด" มีบรรทัด prompt บอก AI ตรง ๆ (ห้ามแต่งเติม / ยังไม่คำนวณ / notAvailable) — **ไม่มีการ degrade เงียบ** สักจุดเดียว engine ปิดปากตัวเองเสมอเมื่อไม่มีของ (ดูหัวข้อ "ยืนยัน" ท้ายเอกสาร)

## กลุ่ม (ก) — หายเมื่อ TNP r391 (`witteTransneptunianPositions`) ลง

flag #1–#5 คือ "ก้อนเดียวกัน" 5 จุด: TNP ของ Witte (Cupido/Hades/Kronos/Zeus) เฟส 1 ไม่มีพิกัด engine เลย (position = null) จึง:
- packet ยัด `notAvailable:"witteTransneptunianPositions"` + `tnpPositionSource:"...not_wired_phase1"` (2 ชั้น: natal + auslosung)
- render สั่ง AI: อ้างได้แค่ "ความหมาย" หมวด H (แต่งงาน=Cupido / โรค-ตาย=Hades / ผู้ปกครอง=Kronos / สงคราม=Zeus) — **ห้ามระบุองศา/ราศี/จุดกึ่งกลาง**
- Auslösung: TNP ใช้เป็นตัวกระตุ้น/directed ไม่ได้

**ผลกระทบต่อคำอ่านตอนนี้:** ภาพดาว/จุดไว/แกนสมมาตรทั้งหมดคำนวณจากดาวจริง 10 ดวงเท่านั้น — TNP หายไปจากเรขาคณิตทั้งหมด แม้ Witte ถือ TNP เป็นแกนกลางของสำนัก Hamburg (เช่น Hades = แกนของ Krankheit/Tod, Kronos = แกน Autorität) → คำอ่านปัจจุบัน **ขาดมิติ TNP ที่เป็นหัวใจของศาสตร์นี้**

**หลัง r391 ลง (TNP มีพิกัดจริง):** flag #1–#5 เคลียร์ทันที → TNP เข้าร่วม Halbsumme/Planetenbild/sensitive Punkte/Auslösung ได้เต็ม → คำอ่านครบมิติสำนัก Hamburg
**มูลค่า:** สูงสุดในบรรดาทุก gap — เปิดแกนตีความหลักของ Witte (บท 19/27/39/40) ที่ตอนนี้อ้างได้แค่ "ความหมายลอย ๆ" ไม่มีตำแหน่งในดวงจริง

## กลุ่ม (ข) — หายไม่ได้ถ้า user ไม่ให้เวลาเกิด (Meridian / Asc / Moon)

flag #6–#11 — ขึ้นกับ **input ของ user** ไม่ใช่ implementation → สร้างโค้ดเพิ่มเท่าไรก็ไม่หาย ถ้าไม่มีเวลาเกิดก็คือไม่มีจริง ๆ:
- ไม่มีเวลา → ไม่มี Meridian(MC)/Aszendent → ตัด `meridianPictures`, `meridianSensitivePoints`, `prog_mc` (Meridianเคลื่อน) ทั้งหมด
- จันทร์ติดธง `uncertain` (จันทร์วิ่ง ~13°/วัน → เที่ยงวันคลาดได้หลายองศา)
- `birthTimeMode:"unknown_noon_anchor"` (ยึดเที่ยงวัน)

**ผลกระทบต่อคำอ่าน:** Witte ถือ Meridian = "แกนกระตุ้นระดับนาที" (บท 07/33/36/38) และ Asc = ตัวตน/ร่างกาย → ไม่มีเวลา = อ่านได้เฉพาะครึ่งผลรวมดาว-ดาว, จับเวลาได้แค่ดาวจร/ส่วนโค้งอาทิตย์/☉☽เคลื่อน (ไม่มี Meridianเคลื่อนที่แม่นระดับนาที)
**สถานะ:** ถูกต้องแล้ว — เป็น degrade ที่ควรมี · **ไม่ใช่ของที่ต้องไปเติม** (เว้นแต่ทำ rectification = เดาเวลาเกิด ซึ่งเป็นคนละฟีเจอร์ · คัมภีร์บท 11/11a/18 "unbekannte Geburtszeit" = roadmap แยก)

## กลุ่ม (ค) — ยังขาดจริง = ต้อง implement/หาแหล่งเพิ่ม (r391 + เวลาเกิด ก็ไม่ช่วย)

นี่คือส่วนที่ผู้ตรวจต้องเน้น — สิ่งที่ยัง "ขาดจริง" แม้ TNP ลงแล้วและ user ให้เวลาเกิดครบ:

### ค-1 · orb ของ antiscia + parallel = "วิธีสากล" ไม่ใช่ค่า Witte (flag #14)
`ORB_ANTISCIA_DEG=1.0` / `ORB_PARALLEL_DEG=1.0` — โค้ดประกาศตรง ๆ ว่า Witte จัด antiscia เป็น "จุดไวหมวดแรกสุด/Anareta" (บท 16/36) และ parallel เป็นลายเซ็นเดคลิเนชัน (บท 03/23/46) **แต่ไม่เคยระบุตัวเลข orb** → ระบบยืม orb 1° จาก Ptolemy PD (zone5)
- **ผลกระทบ:** เกณฑ์ตัด/รับคู่ antiscia+parallel ไม่ได้อิงตัวเลข Witte จริง อาจกว้าง/แคบผิดสำนัก → ชุดคู่ที่โผล่ในคำอ่านอาจไม่ตรงที่ Witte เอง fandhong
- **ต้องเติม:** ขุดคัมภีร์ Witte verbatim หา orb ของ Spiegelpunkte/Deklinationsparallele (ถ้ามี) หรือประกาศ policy ว่า "Witte ไม่ให้ → คงวิธีสากล" อย่างเป็นทางการ
- **มูลค่า:** กลาง — เป็น sourcing fidelity (ตรงกับกฎ "คัมภีร์ = source of truth ห้าม fit")

### ค-2 · ตัวเลของศาตัวอย่างจากคัมภีร์ = OCR Fraktur เพี้ยน (flag #15)
กฎ 8 เตือน AI ว่าองศาในตัวอย่างบท 12/30/31/44 มาจาก OCR อักษร Fraktur → ต้องเทียบสแกน
- **ผลกระทบ:** ถ้า AI ยกตัวเลของศาจากตัวอย่างมาสอน/อ้าง อาจผิดจากต้นฉบับ
- **ต้องเติม:** transcribe verbatim สะอาดจาก r2a/r2c PDF (มีไฟล์ครบ 46 บทในโฟลเดอร์แล้ว) → ลบความไม่แน่นอนนี้
- **มูลค่า:** กลาง — data quality ของคัมภีร์อ้างอิง

### ค-3 · พจนานุกรม "การอ่านเชิงวิธี" ยังไม่ครบทุกคู่ (flag #16, #17)
กฎ 4: "คู่ที่พจนานุกรม (`11-method-reading-uranian.md`) ยังไม่มีจริง ๆ จึงอธิบายเชิงเรขาคณิต" → คู่ดาวที่ไม่อยู่ทั้งใน Witte verbatim และพจนานุกรม = AI อ่านได้แค่รูปเรขาคณิต ("แกนสมมาตร A/B") ไม่มีเนื้อความหมาย
- **ผลกระทบ:** คู่ที่ตกช่องว่างนี้ = คำอ่านแห้ง ไม่มีธีมชีวิต
- **ต้องเติม:** ขยายพจนานุกรมภาพดาวให้ครอบคู่ที่เหลือ (โดยไม่ลอก Ebertin/Brummund/Niggemann ที่มีลิขสิทธิ์ — ต้องเรียบเรียงเองจาก Witte PD + ความหมายดาวสาธารณะ)
- **มูลค่า:** สูง — เพิ่ม coverage คำอ่านโดยตรง (ยิ่ง TNP ลง คู่ยิ่งเพิ่ม ยิ่งต้องการพจนานุกรมครบ)

### ค-4 · ไม่มีระบบเรือน (Häuser) เลย แม้มีเวลาเกิด (flag #7 · "houses")
`forbiddenFieldsWhenNoTime` ใส่ `"houses"` ไว้ — แต่ตรวจ engine แล้ว **ไม่มีการคำนวณเรือนเลยแม้แต่ตอนมีเวลาเกิด** (engine มีแค่ MC/Asc เป็นจุด ไม่มี house cusp) ทั้งที่คัมภีร์มีบทเรือนชัดเจน: บท 32 (Häuser des Aszendenten), 38 (Häuser des Geburtsmeridians), 41 (Häuser der Planeten)
- **ผลกระทบ:** เทคนิคเรือนของ Witte (บ้านของ Asc/Meridian/ดาว) หายทั้งหมด — flag "houses" จึงเป็น false comfort (สั่งห้ามเมื่อไม่มีเวลา ทั้งที่ไม่เคยมีให้ห้ามตั้งแต่แรก)
- **ต้องเติม:** implement house engine ตามบท 32/38/41
- **มูลค่า:** กลาง-สูง — เป็นเทคนิค Witte แท้ที่ยังไม่แตะเลย

### ค-5 · จันทร์จร (transitMoon) ตัดจงใจ — ยังไม่มีชั้นเดือน/วันจากจันทร์จริง (flag #18)
Auslösung ตัด transit Moon ออก เพราะ snapshot เที่ยงวัน = สุ่ม แล้วชดเชยด้วย ☽progressed (ชั้นเดือน)
- **ผลกระทบ:** เหตุการณ์ที่ผูกกับจันทร์จรจริง (Witte บท 14/15/43: Synodische Lunation / Profektion+Lunation / Lunarhoroskop eines Tages) จับไม่ได้ — ชั้นวัน/ชั่วโมงจากจันทร์หายไป
- **ต้องเติม:** จันทร์จรแบบ time-resolved (คำนวณเวลาในวันที่แตะ ไม่ใช่ snapshot เที่ยงวัน) หรือ implement Lunation/Profektion (บท 14/15/43)
- **มูลค่า:** สูง — เปิด "จับวัน/ชั่วโมง" ที่ตอนนี้ได้แค่ระดับเดือน (เทียบกับ DaySniper ปาจื้อที่ยิงวันได้)

### ค-6 · notAvailable / forbiddenFields ไม่ถูก render เข้า prompt เป็นข้อความ (ช่องโหว่เชิงกลไก)
`render.ts` **ไม่อ่าน** `packet.notAvailable` และ `packet.forbiddenFieldsWhenNoTime` เลย — มันสร้างประโยคเตือนของตัวเอง (กฎ 6, บล็อก TNP, บรรทัด "ไม่มี Meridian") ซ้อนกันอยู่แล้ว จึง **ยังครอบคลุมครบ** แต่ถ้าอนาคตเพิ่ม field ลง `notAvailable` โดยไม่แก้ prompt ด้วยมือ → จะ degrade เงียบ
- **ผลกระทบตอนนี้:** ไม่มี (ครอบคลุมด้วย prose แล้ว)
- **ต้องเติม (กันเหนียว):** ให้ render วน `packet.notAvailable`/`forbiddenFieldsWhenNoTime` ออกมาเป็นบรรทัดอัตโนมัติ → single source of truth ป้องกัน drift
- **มูลค่า:** ต่ำ-กลาง — hardening ความซื่อสัตย์ระยะยาว

## "หลัง r391 เหลือขาดอะไร" — สรุปชัด ๆ

r391 (TNP ephemeris) เคลียร์เฉพาะ **กลุ่ม (ก)** = flag #1–#5 เท่านั้น

**หลัง r391 + user ให้เวลาเกิดครบ ยังเหลือขาดจริง (กลุ่ม ค):**
1. **orb antiscia/parallel ไม่ใช่ค่า Witte** (ค-1) — sourcing
2. **องศาตัวอย่างคัมภีร์ยัง OCR เพี้ยน** (ค-2) — data quality
3. **พจนานุกรมภาพดาวยังไม่ครบทุกคู่** (ค-3) — coverage · **มูลค่าสูง**
4. **ไม่มีระบบเรือน Häuser เลย** (ค-4) — เทคนิค Witte บท 32/38/41 · **มูลค่ากลาง-สูง**
5. **ไม่มีจันทร์จร/Lunation/Profektion** (ค-5) — จับวัน-ชั่วโมง บท 14/15/43 · **มูลค่าสูง**
6. **notAvailable ไม่ auto-render เข้า prompt** (ค-6) — hardening กัน degrade เงียบ

**และไม่มีวันหาย (โดยเจตนา ถูกแล้ว):**
- กลุ่ม (ข) Meridian/Asc/Moon เมื่อไม่มีเวลาเกิด — เป็น degrade ตาม input ที่ควรมี
- Apollon/Admetos/Vulkanus/Poseidon (Sieggrün) — ตัดถาวรเพราะลิขสิทธิ์ (flag #19)

## ยืนยัน: ไม่มีการ "บอกตรง" ที่ผิด — engine ซื่อสัตย์ทุกจุด

ตรวจครบ 4 ไฟล์แล้ว **ทุก flag ที่ขาด มีข้อความสั่ง AI ให้ยับยั้งอย่างชัดเจน ไม่มี degrade เงียบ:**
- ของที่ยังไม่คำนวณ (TNP) → `notAvailable` + "⛔ ห้ามระบุองศา/ห้ามใช้เป็นตัวกระตุ้น" ทั้ง natal + ชั้นเวลา
- ของที่ engine ไม่เจอในดวงจริง (ภาพดาว/จุดไว/จุดกระจก/เดคลิเนชัน/การกระตุ้น) → ทุกบล็อกมี "ไม่พบ…ภายใน orb → ห้ามสร้าง…เองจากองศา / อย่าแต่งวัน"
- ของที่ไม่ใช่ Witte verbatim (orb สากล + พจนานุกรมสังเคราะห์) → ประกาศป้าย "วิธีสากล" / "การอ่านเชิงวิธี ไม่ใช่ verbatim Witte" (3 ภาษา)
- ของที่คลาด (จันทร์ไม่มีเวลา · true vs mean node · OCR Fraktur) → ป้าย ⚠️ + "ค่าโดยประมาณต้องเทียบสแกน"

จุดเดียวที่เป็น "ความเสี่ยงเชิงกลไก" (ไม่ใช่บั๊กปัจจุบัน) = ค-6: `render.ts` เขียน prompt เตือนด้วยมือ ไม่ได้อ่านจาก `packet.notAvailable` → ถ้าเพิ่ม field ใหม่ลง notAvailable โดยลืมแก้ prompt จะเป็นช่องให้ degrade เงียบในอนาคต ควร wire ให้ render วน array นี้ออกมาเอง
