## 0 — ของที่ „มีอยู่แล้ว" (baseline · เพื่อให้เห็นช่องว่างชัด)

**engine.ts (deterministic):**

- `sensitivePoints` = ผลรวม/ผลต่าง ถูกกระตุ้น (orb ≤ 1.0°) เรียงตาม orb · cap 60
- `auslosung.ts` = ชั้นเวลา 3 ชั้น (ดาวจร/ส่วนโค้งอาทิตย์/ทุติยภูมิ ☉☽MC) จับวัน-เดือน

**render.ts → AI:** ส่ง points + pictures (occupant บน pairTh) + sensitive + TNP 4 ดวง + กฎ 8 ข้อ · **เรียงตาม orb ล้วน ไม่จัดกลุ่ม ไม่ถ่วงน้ำหนัก ไม่ผูกหัวข้อ**

**พจนานุกรม 11-method:** ความหมายคู่กลาง + **สมการ 3 ทิศ (A/B=C ⟺ A/C=B ⟺ B/C=A)** + Basic Five + จุดส่วนตัว + 4 TNP → แต่เป็น „คำสั่งอ่าน" ให้ AI ทำเอง ไม่ใช่ค่าที่ engine ปั้นมา

**build-prompt.ts:** มี topic router แล้ว **แต่ที่ชั้นเลือกไฟล์คัมภีร์เท่านั้น** (intent → เลือก section ของ `10`/`11`) · ไม่ได้กรอง/จัดอันดับ „ภาพดาวจริงในดวง" ตามหัวข้อ

**resonance.ts (R1-R6):** ทำ cross-science สำหรับ western/vedic/ziwei/qizheng/bazi — **ยูเรเนียนยังไม่อยู่ในวง** (ตัดจงใจ)

## 2 — จัดอันดับสุดท้าย (ทำก่อน→หลัง)

## 3 — แยกชัด: ทำได้ด้วย „คัมภีร์+engine ที่มี" vs „ต้องเพิ่มของใหม่"

### ✅ ทำได้เลยด้วยของที่มี (เป็นงานจัดระเบียบ/ป้าย ไม่ใช่ดาราศาสตร์ใหม่)
- **A** — engine ส่ง `orbDeg`/`orbArcmin` อยู่แล้ว → เติมชั้นป้าย + narrate
- **B** — `personalPoints` มีใน chart แล้ว → flag ภาพดาว/จุดไวที่ occupant หรือคู่ = จุดส่วนตัว แล้วดันขึ้น/แยกกลุ่ม (Basic Five มีในพจนานุกรมแล้ว)
- **C** — `halbsummen` เก็บ **ทุกคู่** แล้ว → หา „mid ตรงกัน ±orb" = ภาพ 4 ดาว (a+b=c+d บท 44) · group pictures ที่แชร์แกน = กองซ้อน 5-6 ดาว (บท 31) · สมการ 3 ทิศมีในพจนานุกรมแล้ว (แค่ให้ engine ปั้น permutation ออกมาให้เห็นชัด แทนให้ AI คิดเอง)
- **D (บางส่วน)** — ชุดดาวต่อหัวข้ออ้าง Witte ได้ (บท 42 + Cupido/Hades/Kronos/Zeus) · ตัวคัด pictures ตามดาว = logic ธรรมดา · router ระดับไฟล์คัมภีร์ก็มีแล้ว

### ❌ ต้องเพิ่มของใหม่จริง
- **E** — module synastry ยูเรเนียนใหม่ (composite ☉+☉, MC+MC, ดาว B บนแกน A) + ช่องรับ 2 ดวงในแผง · **คัมภีร์บท 42 หนุนเต็ม → เพิ่มได้แบบ PD สะอาด**
- **F** — ต้องนิยาม „ธาตุ/polarity ของภาพดาวยูเรเนียน" (ไม่มีในคัมภีร์) + ดึงยูเรเนียนเข้า resonance.ts · **ส่วนที่ไม่มีคัมภีร์ = ต้องเขียนเป็นสังเคราะห์ล้วน ติดป้ายเข้ม**
- **D (เติมเต็ม)** — ตาราง „หัวข้อ→ชุดดาว" ที่เจาะจง (เช่น การเงิน=♀/♄/♃/Kronos) เกินที่ Witte ระบุตรง → ส่วนเกินเป็นสังเคราะห์ ต้องอยู่ใน 11-method + ป้าย „การอ่านเชิงวิธี"

## 4 — ข้อควรระวัง (source of truth · เขียนเป็นเงื่อนไข ไม่ใช่ทางเลือก)
1. ชั้น A/B/C/D „จัดระเบียบวัตถุดิบเดิม" — **ห้ามเปลี่ยนค่าดาราศาสตร์/orb/ความหมาย** · ทำได้เพราะเป็น presentation + logic คัด ไม่ใช่ fit ตำรา
