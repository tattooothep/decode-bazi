## Product-Chain Rule

A feature is not a stable product. Require a chain:

- 3rd/Mercury = specification, code, API, prompt, documentation,
- MC/10th = product direction and release standard,
- 6th = bugs, QA, repeated engineering work,
- 7th/11th = users, platform, integrations, adoption,
- 4th/12th = infrastructure, backend, data asset, hidden maintenance,
- 2nd/Fortune/8th = build cost, runway, security/privacy liability.

If Mercury/3rd is strong but 4th/12th are weak, the idea is clear but the system is unstable. If MC/10th is strong but 6th is weak, roadmap is good but engineering execution is thin. If 7th/11th are strong but 2nd/8th are weak, users want it but infrastructure, token, cloud, or liability cost can break the project.
