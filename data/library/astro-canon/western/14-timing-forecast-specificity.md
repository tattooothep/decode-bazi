# Western Timing, Forecast, And Event-Window Specificity Pack

License mode: summary-only operational rule pack. Use for Hourkey runtime readings only. Do not expose raw OCR, licensed source prose, page order, or private source file paths.

## Purpose

Use this pack when the user asks about a year, month, date, "when", forecast, transit, return, progression, or turning point in Western astrology.

The answer must use the target reference date supplied by Fusion and the timing fields actually present in the chart packet. Do not invent progressions, returns, directions, profections, horary, electional, or composite charts unless those exact packet fields are supplied.

## Timing Hierarchy

1. Natal promise first: Ascendant/chart ruler when timed, Sun/Moon, MC/10th, relevant house/ruler, sect, dignity, lots, dominant planets, and natal aspects.
2. Active transit next: target-year `transits`, exact `refDate`, transit-to-natal aspects, retrogrades, and return-cycle fields when present.
3. Specialty timing only when supplied: progressions, primary directions, solar arcs, annual profections, solar/lunar returns, eclipse contacts, horary, electional, or relocation packets.
4. Topic confirmation: connect timing to the asked house and significators, not to generic planet meanings.
5. No-time guard: when birth time is missing, do not use houses, angles, timed lots, or house-overlay timing unless the packet provides a no-time substitute.

## Forecast Rules

- A timing claim is strong when natal promise, topic house/ruler, and current transit/return field all point in the same direction.
- A single transit is not enough for a life-changing verdict unless it hits an angle, luminary, chart ruler, MC, Lot, or the exact asked-topic ruler.
- If the user asks a future year, use the target-year refDate and do not answer from the present-year transit if the packet has been recomputed.
- If the user asks a month/day but the packet has only one reference date, say the year/near-date tendency is readable but a full monthly timeline is not proved.
- Return-cycle fields describe broad phase pressure; they should not replace exact transit/aspect evidence.

## Output Contract

For timing/forecast answers, include:

- one natal promise line naming Asc/chart ruler or no-time substitute,
- one asked-topic house/ruler/significator line,
- one transit/refDate/return-cycle line when supplied,
- one specialty-timing availability line if progressions/returns/directions are requested,
- one precision line saying exact date, target-year snapshot, year-only, or no-time limited,
- one decisive forecast verdict for the exact timing question.
