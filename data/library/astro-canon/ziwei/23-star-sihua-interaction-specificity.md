# Zi Wei Star, Palace, Sanfang, And Si Hua Interaction Specificity Pack

License mode: summary-only operational rule pack. Use for Hourkey runtime readings only. Do not expose raw OCR, licensed source prose, page order, or private source file paths.

## Purpose

Use this pack when the user asks about "ปฏิกิริยาดาว", "ดาวกระทบกัน", "ดาวสัมพันธ์กัน", "ดาวชน", "四化 ทำงานยังไง", "飛化", "飛星", "三方四正", or interaction/contact questions in Zi Wei Dou Shu.

Zi Wei interaction is palace and transformation based. Do not answer from one star alone. A valid interaction answer must name the star/palace, 四化 or 三方四正 channel, support/friction, and timing layer if relevant.

## Interaction Chain

1. Name the star/palace on each side: 命宮, 身宮, asked palace, opposite palace, 三方 palace, or pair-packet palace.
2. Name the interaction channel: same palace, 對宮, 三方四正, 生年四化, 大限四化, 流年/流月/流日四化, annual stars, or closed pair packet.
3. Judge star condition: 主星 brightness, 吉曜 support, 煞曜/空劫/刑耗 pressure, and whether the star belongs to the asked topic.
4. Judge 四化 direction: 化祿 opens/inflow, 化權 pushes/control, 化科 clarifies/reputation/documents, 化忌 knots/attachment/leakage.
5. Bind to the user topic and timing. Do not let one 化祿 erase repeated 忌/煞, and do not let one 忌 erase strong repeated support.

## Contact Weighting

- 命宮/身宮 + asked palace interaction is the strongest single-chart channel.
- Asked palace + 三方四正 repetition is stronger than one isolated auxiliary star.
- 大限/流年 repeats a natal theme: stronger timing signal.
- 流月/流日 only triggers short windows after natal + 大限/流年 promise exists.
- Pair mode must use `PAIR_INTERACTION_PACKET ziwei` only for cross-person claims.

## Output Contract

For star/四化 interaction answers, include:

- one interaction line naming both palace/star sides,
- one channel line naming same palace/對宮/三方四正/四化/timing/pair source,
- one support-versus-friction line naming 化祿/權/科/忌 or 吉曜/煞曜,
- one timing line if 大限/流年/流月/流日 is relevant or unavailable,
- one concrete topic consequence,
- one guard line if the requested contact is not in the packet.
