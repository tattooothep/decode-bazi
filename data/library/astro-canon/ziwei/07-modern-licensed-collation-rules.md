## What This Adds

1. answer the asked topic, not a generic whole-chart essay,
2. treat each palace as topic-specific; a palace can be weak for one question and usable for another,
3. require a chain from natal promise to timing trigger before making strong event claims,

## Evidence Chain

For a strong verdict, ask:

1. Is the natal topic palace clear?
2. Does 三方四正 support or contradict it?
3. Does 生年四化 create a lifelong pattern touching this topic?
4. Does 大限 repeat the topic?
5. Does 流年 repeat or trigger it?
6. If the question is short-term, do 流月/流日 support the same direction?

Verdict strength:
- Strong: topic palace + 三方四正 + 大限/流年 repeat.
- Medium: topic palace is clear but timing is mixed.
- Weak: only a minor star, romantic star, or one-day trigger supports it.

## 飛星 / 四化 Direction

Read transformations as movement:

- From which stem/layer does the transformation arise: 生年, 大限, 流年, 流月, 流日, or pair crossSiHua.
- Which star transforms.
- Which palace receives it.
- Which life topic that receiving palace controls.

Plain rule:
- 化祿 brings desire/flow/resource to the receiving palace.
- 化權 brings pressure/action/control to the receiving palace.
- 化科 brings protection/name/document/order to the receiving palace.
- 化忌 brings attachment/knot/debt/worry/blockage to the receiving palace.

Never flatten them into one score. A 化祿 to a risky palace can increase appetite or entanglement; a 化忌 to a protected palace can become work to manage, not instant disaster.

## Relationship And Pair Reading

Single chart:
- 夫妻 alone is not enough. Read 夫妻 + 對宮 + 三方四正 + 命/身 temperament + 福德 emotional comfort.
- 紅鸞/天喜 are triggers, not proof of marriage.
- 化忌 in relationship topics means an emotional knot, duty, debt, or repeating conflict; name the actual knot.

Pair chart:
- Use only the pair packet; do not invent extra cross-palace overlays.
- Good pair evidence requires at least one life-palace/spouse-palace connection plus supportive crossSiHua.
- If one person receives 化祿/科 and the other receives 化忌/權 pressure, say plainly that support is uneven.
- A strong attraction pattern with repeated 忌/煞 is "ผูกกันแรงแต่กดดัน", not automatically stable.

## Timing

Do not answer future years from natal promise alone if the packet supplies timing layers.

For timing questions:
- name 大限 first for the life stage,
- name 流年 for the year focus,
- use 流月/流日 only when supplied,
- if only year data exists, stop at year-level,
- if month/day data exists, give a practical action window but avoid guaranteed fate.

Short-term timing is valid only when it confirms the broader layer. A good day inside a bad month/year is usable for small actions, not for declaring a life-changing success.
