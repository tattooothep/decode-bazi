# directive โหมด "อ่านทั้งชีวิต" (ร่าง งาน 3 · แก้ตามผู้ตรวจ 4 · ยังไม่ wire · additive — ไม่แตะโหมด Q&A เดิม)

> ปัญหาที่แก้ (จากวิจัย 22 ดวง): ระบบผูกทุกคำอ่านกับ refDate ปีปัจจุบัน → ดวงประวัติศาสตร์ถูกฉายเป็นคนเป็นอายุ 100+ · จุดเปลี่ยนชีวิตถูกตอบเป็น "ปีนี้-ปีหน้า" แทนภาพทั้งเส้น
> สวิตช์: `wholeLifeMode?: "research" | "user"` (แบบ bookMode ของ r391 · default undefined = โหมด Q&A เดิม byte-identical)
> ข้อ 1-2, 6-7 ใช้ร่วมทั้งสองโหมด · ข้อ 3-5 ต่างตามโหมด
> ⚠️ เงื่อนไขก่อน wire (จากผู้ตรวจ 4): ต้องทดสอบด่าน fact-claim (r508) กับคำตอบโหมดนี้อย่างน้อย 3 ดวง — โหมดนี้พ่นคู่ ปี↔大運 ทั้งชีวิต ต้อง verify ว่า packet ส่งตาราง大運ครบทุกช่วง (รวมอดีต) ให้ gate ตรวจได้ ไม่งั้นจะโดน 502

## ข้อความ directive (ที่จะเข้า prompt จริง)

=== โหมดอ่านทั้งชีวิต (ไม่ผูกปีปัจจุบัน) ===
1. ห้ามยึดปีปัจจุบัน/วันอ้างอิงจรเป็นศูนย์กลางคำตอบ — ให้ไล่อ่าน "แผนที่ทั้งชีวิต" จากชั้นเวลาเชิงคณิตที่ engine ส่งมา (ปาจื้อ: ตาราง大運ทุกช่วง · จื่อเวย: 大限ทุกเรือน · เวท: มหาทศาทุกตัว · ดาวจริง: 行限ทุกช่วง · ตะวันตก/ยูเรเนียน: รอบดาวช้าและส่วนโค้งที่ให้มา — **หมายเหตุ: สองศาสตร์นี้ packet ปัจจุบันส่งชั้นจรเฉพาะปีเป้าหมาย ให้อ่านภาพชีวิตจากรอบ Jupiter/Saturn return ที่มี และแจ้งข้อจำกัดตรงๆ ห้ามแต่งไทม์ไลน์เต็มเอง**) ตั้งแต่เกิดจนสุดตารางที่ส่งมา
2. ทุกช่วงวัยให้ระบุ: ช่วงอายุ (ตัวเลข) · ธีมหลัก · ขึ้นหรือลง · เหตุผลจากผัง (ดาว/ธาตุ/เรือนที่คุมช่วงนั้น)
3. หน้าที่บังคับ: ชี้ "ด่านวิกฤต" 1-3 ด่านที่หนักที่สุดของทั้งชีวิต**ตามหลักฐานจริง** — ช่วงวัยที่ชั้นเวลาหลายชั้นกดพร้อมกัน พร้อมประเภทความเสี่ยง (กาย/ใจ/อุบัติเหตุ/ทรัพย์/คดี) และระบุความหนาของหลักฐาน (กี่ชั้นกดพร้อมกัน) · กล้าฟันเรื่องหนักตรงไปตรงมา แต่**ทุกด่านต้องจบด้วย "ทางรับมือ" อย่างน้อย 1 ข้อ** — ด่านวิกฤตคือแผนที่ไว้เตรียมตัว ไม่ใช่คำพิพากษา · **ห้ามแต่งด่านเพิ่มเพื่อให้ครบจำนวนถ้าผังชี้ไม่ชัด** · **ห้ามเลื่อนด่านไปวัยชราเพื่อความปลอดภัย — ถ้าชั้นเวลาซ้อนกดหนักสุดตกวัยหนุ่ม (ก่อน 50) ต้องระบุตามนั้นตรงๆ**
4. เรื่องอายุขัย: **ห้ามการันตีหรือใบ้ปีตาย** · ต้องชี้ช่วงวัยที่ผังเตือนแรงสุดเรื่องกาย ด้วยถ้อยคำ "สัญญาณ" เสมอ เช่น "ช่วงอายุ X-Y คือช่วงที่ผังเตือนเรื่องร่างกายแรงสุด — ควรตรวจสุขภาพ/เลี่ยงความเสี่ยง Z ล่วงหน้า" · การไม่ชี้ช่วงเลย = ผิดหน้าที่ · การชี้แบบคำตัดสินตายตัว = ผิดหน้าที่เท่ากัน · ข้อ 3-4 อยู่ใต้ "นโยบายคำตอบผู้ใช้จริง" เสมอ (สุขภาพ=สัญญาณ ไม่ใช่คำสั่งแทนแพทย์)
5. สถานะเจ้าชะตา แยกตามโหมด:
   - **[research]** อย่าสมมติสถานะมีชีวิต/เสียชีวิต — อ่านเป็นแผนที่ตามผังล้วน เป็นกลางต่อสถานะ
   - **[user]** ถือว่าเจ้าชะตามีชีวิตอยู่ ณ อายุปัจจุบันจาก refDate: ด่านที่ผ่านมาแล้วเขียนเป็นอดีต ("ด่านช่วงอายุ X-Y ผ่านมาแล้ว — ตรวจสอบได้ว่าตรงไหม") · ด่านข้างหน้าเขียนเป็นสัญญาณ+การเตรียมตัว
6. ปิดท้ายด้วยสรุปเส้นชีวิต 3-5 บรรทัด: ทศวรรษทอง · ทศวรรษเสี่ยง · จุดพลิกใหญ่สุด (ช่วงอายุตามที่ผังชี้ได้จริง)
7. กติกาเดิมคงทั้งหมด: ห้ามเดาตำแหน่งดาว ห้ามใช้ความรู้นอกผัง ทุกคำฟันธงอ้างหลักฐานผัง ห้ามใส่ % ความแม่น

## precedence (จากผู้ตรวจ 5)
- `bookMode && wholeLifeMode` พร้อมกัน: **wholeLifeMode ชนะฝั่ง timing** (สลับบล็อกจังหวะเวลา) · bookMode คุมฝั่ง directive คำถามตามเดิม — สองตัวแตะคนละบล็อก ไม่ชนกัน
- ห้าม implement เป็น question-replacement (canon router จะหลุด) — swap เฉพาะบล็อก `=== จังหวะเวลาที่ใช้คำนวณจร ===` จุดเดียว

## จุด wire ที่วางแผน (ยังไม่ทำ — รอครบ 5 ลายเซ็น)
- `buildSciencePrompt(..., opts: { wholeLifeMode?: "research" | "user" })` — สลับบรรทัด timing เหมือน bookMode ของ r391 (additive · default undefined = Q&A เดิม byte-identical)
