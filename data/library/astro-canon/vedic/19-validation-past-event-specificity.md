## Purpose

Use this pack when the user asks "ตรงไหม", "ทำไมไม่ตรง", "พิสูจน์ดวง", "ที่ผ่านมา", "อดีต", "เคยเกิดอะไร", "ช่วงก่อน", "validate", "calibrate", "life events", or wants the reading to prove chart specificity in Jyotish.

The answer must be concrete and testable from the supplied Lagna/Moon/graha/bhava/dasha packet. Do not give generic nakshatra or rashi-only statements when stronger packet evidence exists.

## Validation Chain

1. Start with Lagna/Lagnesha when birth time exists; otherwise use Moon, Moon nakshatra, Sun, graha signs, and no-time safeguards.
2. Add topic bhava and karaka: 10th/D10 for career, 2nd/11th/5th/9th for money, 7th/Venus/Jupiter/D9 for relationship, 6th/8th/12th for health/risk, 4th/9th/12th for home/travel.
3. Add varga, shadbala, ashtakavarga, drishti, and yoga only when supplied.
4. Add timing only from supplied Vimshottari mahadasha/antardasha, sub-periods, gochara, and ashtakavarga. Do not invent alternate dasha or exact dates.
5. Mark confidence:
   - high: natal promise + bhava/karaka + dasha/gocara repeat the same theme,
   - medium: bhava/karaka/varga evidence is clear but timing is broad,
   - low: single graha/nakshatra signal only.

## Event Marker Rules

- Career marker: 10th lord, D10, Saturn/Sun/Mercury/Mars, dasha lord, gochara trigger.
- Money marker: 2nd/11th/5th/9th, Jupiter/Venus/Mercury/Rahu, gains versus debt/speculation split.
- Relationship marker: 7th lord, Venus/Jupiter, D9, Moon/Tara/pair packet when supplied.
- Health/routine marker: 6th/8th/12th, Moon/Saturn/Mars, dasha/gocara pressure.
- Home/travel marker: 4th/9th/12th, Moon/Jupiter/Rahu/Ketu, dasha/gocara activation.

## Output Contract

For validation/past-event answers, include:

- one core-signature line naming Lagna/Lagnesha or no-time Moon substitute,
- three concrete life-event markers tied to exact bhava/graha/varga/dasha/gocara evidence,
- confidence label for each marker: high / medium / low,
- one "what would confirm or falsify this" line,
- one precision-limit line naming missing birth time/dasha/month/day/pair data if unavailable,
- no generic trait list unless each trait is tied to a packet anchor.
