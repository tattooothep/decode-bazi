# Jyotish Graha Interaction, Drishti, Yoga, And Contact Specificity Pack

License mode: summary-only operational rule pack. Use for Hourkey runtime readings only. Do not expose raw OCR, licensed source prose, page order, or private source file paths.

## Purpose

Use this pack when the user asks about "ปฏิกิริยาดาว", "ดาวกระทบกัน", "ดาวสัมพันธ์กัน", "ดาวชน", "drishti", "aspect", "yoga", "graha contact", "conjunction", or interaction/contact questions in Jyotish.

Jyotish interaction must be read through graha role, bhava, rashi lordship, drishti, dignity, varga, bala, dasha, and gochara. Do not answer from natural graha meaning alone.

## Interaction Chain

1. Name each graha and chart role: Lagnesha, Moon lord, bhava lord, karaka, dasha lord, gochara trigger, or pair-packet graha.
2. Name the contact channel: conjunction/same rashi, Parashari drishti, house relationship, rashi lord/dispositor relation, yoga candidate, varga repetition, dasha-gocara activation, or closed pair packet contact.
3. Judge capacity: dignity, friend/enemy relation, combustion, retrograde, dusthana/maraka tie, shadbala, ashtakavarga, and varga confirmation when supplied.
4. Judge direction: benefic support, malefic pressure, functional lordship, dusthana pressure, upachaya growth, or dharma/artha/kama/moksha topic.
5. Bind to the user topic and timing.

## Contact Weighting

- Lagnesha/Moon/dasha lord contact outweighs a generic graha meaning.
- 10th/D10, 7th/D9, 2nd/11th, 6th/8th/12th topic contacts must be read with the topic bhava.
- Yoga candidates require cancellation checks; do not declare a yoga fully operative from one condition.
- Dasha activates the contact; gochara triggers or pressures it.
- Pair mode must use `PAIR_INTERACTION_PACKET vedic` only for cross-chart claims.

## Output Contract

For graha interaction answers, include:

- one contact line naming both grahas and contact channel,
- one role line naming Lagna/Moon/bhava/karaka/dasha role,
- one capacity line naming dignity/varga/bala/ashtakavarga or unavailable,
- one support-versus-pressure line naming benefic/malefic/functional/dusthana factors,
- one concrete topic consequence,
- one guard line if the requested contact/yoga is unsupported or not in the packet.
