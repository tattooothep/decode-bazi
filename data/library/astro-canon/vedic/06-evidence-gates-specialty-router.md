# Jyotish Evidence Gates And Specialty Router

> Runtime derived summary for Fusion Vedic/Jyotish. This is an operational gate pack, not a raw source pack. It applies the retained public-domain and licensed-internal corpus without quoting or exposing source text.

## Source Scope

- Core natal: BPHS, Brihat Jataka/commentary witnesses, Saravali, Phaladeepika, Jataka Parijata, Uttara Kalamrita, Laghu/Madhya Parashari, Sarvartha Chintamani, Hora Sara, Bhava Kutuhala/Prakasha, Yavana/Kamadhenu/Hora Ratnam-style collation.
- Timing: Vimshottari dasha, BPHS dasha-phala, Laghu-Parashari, Graha Dasha Phalam, Ashtottari witnesses, gochara and ashtakavarga witnesses.
- Specialty witnesses: Prashna Marga/Tantra/Manorama/Ratna/Skandhasthi/Shatpanchasika/Bhrigu prashna; Muhurta Chintamani/Martanda/Ratna/Darpana/Sindhu/Ganapati/Kalaprakashika/Samaya Shuddhi/Vivaha; Tajika Nilakanthi and Varsha Phala witnesses; Jaimini Sutra witnesses; Siddhanta/Vedanga calculation witnesses.
- Licensed/internal materials are used only as extracted operational summaries. Do not quote, cite, reproduce, or expose raw source text, page order, or file paths.

## First Route The Question

Classify the user question before reading:

1. **Natal / life topic**: use Lagna, Moon, Sun, graha, bhava/lord, dignity, drishti, yoga, varga, shadbala, ashtakavarga, dasha, gochara.
2. **Pair / compatibility**: use both natal charts and `PAIR_INTERACTION_PACKET vedic`; do not invent cross-chart contacts.
3. **Future/timing**: use Vimshottari mahadasha/antardasha plus gochara/ashtakavarga/varga confirmation.
4. **Prashna**: requires question-time chart fields. Natal-only packets are not prashna.
5. **Muhurta / electional**: requires panchanga/electional date-time fields, action class, and candidate lagna/moon fields.
6. **Tajika / Varshaphala**: requires annual chart fields such as annual Lagna, Muntha, annual lord, sahama, Tajika aspects/yogas.
7. **Jaimini / Chara**: requires karaka/karakamsa/chara dasha fields. Do not infer them from ordinary natal packet.
8. **Alternate dashas**: Ashtottari, Kalachakra, Chara, Yogini, Naisargika, or prashna dasha require explicit packet fields.
9. **No-time birth**: use only allowed no-time fields; close Lagna, bhava, house lordship, D9/D10 houses, and exact dasha if not supplied or flagged uncertain.

If a specialty branch is missing, mention that one missing layer briefly and still answer from the valid natal/dasha/gochara layer.

## Evidence Chain For Natal Topics

Every natal topic answer should use this chain:

1. Lagna/Lagnesha and Moon/nakshatra baseline.
2. Topic bhava and topic lord.
3. Natural karaka for the topic.
4. Dignity, combustion, retrograde, drishti, dusthana/maraka/upachaya context.
5. Varga support: D9 for maturity/relationship/dharma, D10 for work/status, other vargas only if supplied.
6. Yoga candidates and cancellation/affliction context.
7. Current/target mahadasha and antardasha.
8. Gochara and ashtakavarga confirmation.

Do not let a single yoga, single transit, or shadbala score override a weak topic lord, dusthana pile-up, or dasha contradiction.

## Topic Gates

### Career / Karma

Use 10th bhava/lord, planets in 10th, Lagnesha, Sun, Saturn, Mercury, D10, dasha, and gochara.

Strong career verdict requires repeated support:
- 10th lord strong or connected to Lagna/9th/11th,
- D10 repeats the same planet/topic,
- dasha lord connects to 10th/6th/11th/2nd,
- Saturn/Jupiter/gochara activates the relevant bhava/lord,
- yoga candidate supports authority, recognition, skill, or gains.

Name the work channel: authority, service, trade, analysis, teaching, foreign/backstage, technical/action, creative/beauty, property, finance, research, spiritual/advisory.

### Money / Artha

Use 2nd/11th bhava and lords, 5th/9th fortune/speculation, Jupiter/Venus/Mercury, dhana yoga candidates, dasha, gochara, and ashtakavarga.

Separate:
- earning channel,
- savings/assets,
- gains/network,
- debt/tax/shared money,
- speculation/windfall,
- leakage through 8th/12th.

Rahu in 2nd/11th may show nonlinear gain, foreign/unusual channels, appetite, or volatility. It is not automatic stable wealth.

### Relationship / Marriage

Use 7th bhava/lord, Venus, Jupiter as husband significator where applicable, Moon, D9, 2nd/4th/11th support, dasha, and pair packet in pair mode.

Separate:
- attraction/desire,
- emotional compatibility,
- stability/marriage readiness,
- family/social support,
- timing.

Do not declare marriage/divorce from Venus alone, D9 alone, or one synastry support. Dasha plus 7th/D9 evidence must repeat.

### Health / Risk

Use Lagna/Lagnesha, 6th/8th/12th, Moon, Sun, Saturn, Mars, Rahu/Ketu, dasha, gochara, shadbala/ashtakavarga resilience.

Allowed wording: vitality, workload strain, sleep/routine pressure, chronic/acute risk symbolism, check-up, rest, discipline.

Forbidden wording: diagnosis, fixed disease, death prediction, surgery instruction, medication advice, or replacing medical care.

### Home / Property

Use 4th bhava/lord, Moon/Venus, Mars for land, 2nd/11th funding, 8th/12th hidden costs, dasha and gochara.

Separate acquisition, repair, family burden, relocation, funding, and paperwork. A strong 4th with weak 2nd/11th means desire/root is present but funding needs discipline.

### Foreign / Travel / Isolation

Use 9th/12th/7th/3rd links, Rahu/Saturn, Moon, dasha/gocara. Separate travel, relocation, foreign clients, distance relationship, hospital/rest/isolation, and backstage work.

## Timing Gate

If `vimshottari` exists, do not say dasha is missing.

For year questions:

1. Name the mahadasha and antardasha.
2. State each lord's house/lordship/topic link.
3. Check dignity/varga/shadbala.
4. Use gochara hits and ashtakavarga as confirmation or friction.
5. Do not claim exact month/day unless sub-period or transit-window fields exist.

Mahadasha sets the main field. Antardasha triggers the concrete sub-topic. Gochara opens or pressures the door; it does not become the main event if dasha contradicts it.

## Specialty Packet Gates

### Prashna

Use only when a question-time packet exists. Required fields: question Lagna, Moon, topic house/lord, applying/separating indicators or prashna-specific flags if supplied.

If absent, answer from natal/dasha tendency and say the prashna chart itself is not supplied. Do not turn a birth chart into prashna.

### Muhurta

Use only when electional fields exist: candidate date/time, location, activity class, tithi, nakshatra, weekday, yoga/karana, Moon condition, Lagna/action lord when supplied.

If absent, do not use Muhurta texts to declare a date good/bad. For Fusion natal questions, ignore muhurta unless the user asks for action timing.

### Tajika / Varshaphala

Use only with annual chart fields. If absent, answer future years from natal promise + Vimshottari + gochara and mention annual/Tajika chart is not supplied only when the user asks that layer.

### Jaimini / Karakamsa / Chara

Use only when karaka/karakamsa/chara fields exist. Jaimini witnesses can refine rajayoga/career/spiritual wording only after packet fields are supplied.

### Alternate Dashas

Do not introduce Ashtottari, Kalachakra, Chara, Yogini, Naisargika, or prashna dasha unless packet says that system is available. If only Vimshottari exists, use Vimshottari.

### Siddhanta / Calendar

Siddhanta and Vedanga witnesses are provenance for calculation and calendar discipline. The AI must not recalculate ayanamsa, tithi, nakshatra, or positions from source texts.

## No-Time Mode

If birth time is missing or degraded:

- Use graha in rashi, dignity, retrograde/combustion if supplied, Moon/nakshatra only if stable, and broad graha interactions supplied by packet.
- Do not use Lagna, bhava, house lordship, varga houses, D10 career houses, exact dasha start/end, or marriage/home/health conclusions that need houses unless packet provides a reliable fallback.
- Keep the answer useful by saying what can still be read from Sun/Moon/graha/nakshatra, not by dumping every missing layer.

## Output Contract

Every Vedic answer should include:

1. one Lagna/Moon baseline or no-time substitute,
2. one topic bhava/lord or explicit reason it is unavailable,
3. one karaka/graha strength line,
4. one dasha/gocara line when timing matters,
5. one varga/shadbala/ashtakavarga confirmation when supplied,
6. one practical recommendation tied to the same evidence.

Do not output source audits, readiness percentages, broad textbook definitions, or long missing-data lists in user readings.
