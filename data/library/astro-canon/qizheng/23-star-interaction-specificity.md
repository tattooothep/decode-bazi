# 七政四餘 · Star Interaction, Contact, And Cross-Pressure Specificity Pack

License mode: summary-only operational rule pack. Use for Hourkey runtime readings only. Do not expose raw OCR, source prose, page order, or private source file paths.

## Purpose

Use this pack when the user asks about "ปฏิกิริยาดาว", "ดาวกระทบกัน", "ดาวสัมพันธ์กัน", "ดาวชน", "ดาวหนุน/ดาวกด", "ทำไมดาวนี้ส่งผล", or interaction/contact questions in Qizheng / Guolao.

Qizheng interaction is not a generic Western aspect reading. Use only interactions supplied by the packet: palace placement, same palace, palace lord relation, 三方/對照 when supplied, 恩用仇難 role, 廟旺落陷/升殿 status, 格局, 行限/木土 activation, and pair packet contacts in pair mode.

## Interaction Chain

1. Name each star and its chart role: 命主, 度主, 身主, 恩星, 用星, 仇星, 難星, palace ruler, star in asked palace, or timing trigger.
2. Name the contact channel: same palace, asked palace lord, 三方, 對照, 格局, 行限/限度主, target-year 木土, or closed pair packet.
3. Judge capacity: 廟/旺/升殿 can deliver; 平 is neutral; 落陷/弱 strains delivery.
4. Judge direction: 恩/用 supports, 仇/難 pressures, 命主/度主 makes it personal, 身主 makes it embodied/practical.
5. Bind to the user topic. Never explain star nature without palace/topic/timing evidence.

## Contact Weighting

- 命主 or 度主 contact outweighs ordinary star nature.
- A star that is both 用星 and strong can mitigate pressure but cannot erase a damaged palace alone.
- A star that is 仇/難 and also activates the asked palace/timing is a concrete warning signal.
- Same palace + same topic + timing activation is stronger than a broad 三方 support.
- Pair mode must use `PAIR_INTERACTION_PACKET qizheng` only; do not invent cross-chart star contacts.

## Output Contract

For star-interaction answers, include:

- one interaction line naming both star roles,
- one contact-channel line naming palace/三方/對照/格局/timing/pair source,
- one capacity line naming 廟旺/升殿/平/落陷 or unavailable,
- one support-versus-pressure line naming 恩/用 versus 仇/難,
- one concrete topic consequence,
- one guard line if the requested aspect/contact is not in the packet.
