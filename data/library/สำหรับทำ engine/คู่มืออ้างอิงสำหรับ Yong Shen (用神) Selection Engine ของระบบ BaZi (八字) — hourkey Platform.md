# คู่มืออ้างอิงสำหรับ Yong Shen (用神) Selection Engine ของระบบ BaZi (八字) — hourkey Platform

**TL;DR**
- ตาราง 調候用神 ครบ 120 ช่อง (10 天干 × 12 月) สังเคราะห์จาก《窮通寶鑑》（欄江網）และ「用神提要」ของ徐樂吾 (พิมพ์ครั้งที่ 民國廿六年 / 1937) พร้อม rationale ภาษาจีนต้นฉบับ ใช้เป็น static lookup ขั้นพื้นฐานก่อนปรับด้วย logic อื่น
- Decision tree ลำดับความสำคัญที่ปลอดภัยที่สุดสำหรับ production: **(1) ตรวจ 從格／專旺格／化氣格 → (2) 調候 (เมื่อหนาว/ร้อนสุดขั้ว) → (3) 扶抑 → (4) 通關 → (5) 病藥**; โดย調候 จะ override 扶抑 เมื่อบ้านเดือนหนาว/ร้อนสุดขั้วและไม่มีธาตุปรับสมดุล (อ้างอิงจาก《滴天髓》〈寒暖〉〈燥湿〉)
- การประเมิน 旺衰 ใช้ 3 มิติ (得令／得地／得勢) — โดยอ้างอิงหลักจาก《淵海子平》:「月令者，如象臣行君之令，掌一國之綱紀」 และ《子平真詮·論用神》:「八字用神，專求月令」 น้ำหนักของ月令ในการประเมินที่ใช้กันแพร่หลายในสำนัก 旺衰派 อยู่ที่ประมาณ 50% (ตามที่ระบุไว้ใน suanzhun.net:「月令的影响可以决定日干50%的旺衰结果」) จากนั้นประเมิน根 (通根) ในตาราง藏干 แล้ว合計 stems และ branches; การประเมินผิดด้านนี้คือสาเหตุหลักของ用神ผิด

## SECTION 1 — ตาราง 調候用神 (Climate-Adjustment Useful Gods) ครบ 120 ช่อง

> **หลักการต้นฉบับ (《滴天髓·寒暖／燥濕》)**: 「天道有寒暖，發育萬物，人道得之，不可過也；地道有燥濕，生成品彙，人道得之，不可偏也」
>
> **徐樂吾補註**: 「寒暖燥濕，指全局氣勢言，過猶不及，皆屬偏枯。故八字以中和為貴，調和氣候，為八字中之重要工作也…暖燥太過，喜雨露以潤之；寒濕太過，宜太陽以暄之」
>
> **กฎสำเร็จรูป**: ดวงเกิด亥／子／丑 (สามเดือนหน้าหนาว) ขาด丙火 = 「too cold to use」 ต้องใช้調候ก่อนตัดสินอื่น; ดวงเกิด巳／午／未 (สามเดือนหน้าร้อน) ขาด壬癸水 = 「燥而無生機」 ต้องใช้調候น้ำก่อน
>
> รูปแบบ table: **主用 → 次用 → 再次** ตามลำดับใน「用神提要」 + คำอธิบายต้นฉบับ

## SECTION 2 — Decision Tree การเลือก 用神 (Implementable Logic)

### 2.1 หลักการ Priority Order
จาก徐樂吾《子平真詮評註·論用神》 (民國二十五年, 1936): 「取用之法不一，約略歸納，可分為下列六種：(一)扶抑 (二)病藥 (三)調候 (四)專旺 (五)通關」 — แต่ใน production engine ลำดับการตรวจสอบที่ปลอดภัยที่สุดคือ:

```
1. ตรวจ Special Format (從格／專旺／化氣) ก่อน — ถ้าเข้า, อย่าใช้ 扶抑
2. ตรวจ Extreme Climate (調候 override) — ถ้าหนาว/ร้อนสุดขั้ว, ใช้調候เป็นหลัก
3. ใช้ 扶抑 (default path) — สำหรับ 80% ของดวงปกติ
4. ใช้ 通關 — เมื่อสองธาตุปะทะกันรุนแรง
5. ใช้ 病藥 — refine ขั้นสุดท้าย: หา 病 (ตัวเสียหายในดวง) → ใช้ 藥 (ตัวแก้)
```

    if detect_cong_ge(chart):                 # 從格 (Following)

        # 從旺格: 比劫極旺, no 官殺
        # 從財格: 財星極旺, 日主無根
        # 從殺格: 官殺極旺, 日主無根
        # 從兒格: 食傷極旺
        # 從勢格: 財官食三方均旺, 日主無根
        return follow_dominant_qi(chart)      # 用神 = 旺神 itself + 順生之神

        # 甲己合化土 + 月令辰戌丑未 + 無木根
        return transformed_element(chart)
    
    if detect_zhuan_wang(chart):              # 專旺格 / 一行得氣
        # 曲直(木) 炎上(火) 稼穡(土) 從革(金) 潤下(水)
        return zhuan_wang_yong_shen(chart)
    
    # === STEP 2: Climate Override (調候為急) ===
    # 《滴天髓·寒暖》: 「天道有寒暖…人道得之不可過也」
    # 徐樂吾 (《子平真詮評註·論用神配氣候得失》, 1936): 
    #   「故取用神，於扶抑之外，必須參合氣候，即調候之法也」
    if climate == '過寒':                      # บ้านเดือนหนาว 亥子丑 + ไม่มี丙丁巳午戌未
        primary = QTBJ_TABLE[day_master][month_branch]['primary']
        if primary in chart.available_stems_or_roots():
            return {
                'method': '調候',
                'primary': primary,
                'reason': f'{day_master}生{month_branch}月, 過寒, 非{primary}不暖'
            }
        else:
            return {
                'method': '調候 (缺神)',
                'primary': primary,
                'note': '原局缺調候用神, 需待大運補之, 格局降低'
            }
    
    if climate == '過燥':                      # บ้านเดือนร้อน 巳午未 + ไม่มี壬癸亥子辰丑
        primary = QTBJ_TABLE[day_master][month_branch]['primary']
        return {
            'method': '調候',
            'primary': primary,
            'reason': f'{day_master}生{month_branch}月, 過燥, 必須{primary}潤之'
        }
    
    # === STEP 3: 扶抑用神 (default for 春秋月 or moderate climate) ===
    # 《滴天髓·體用》: 「道有體用，不可以一端論也，要在扶之抑之得其宜」
    if strength in ('偏旺', '旺極'):
        # 旺則泄之 / 克之 / 耗之
        # ลำดับความสำคัญ:
        #   1. 印比เกิน → 用官殺 (克) > 食傷 (泄) > 財 (耗)
        #   2. 印重 → 用財 (破印)
        #   3. 比劫多 → 用官殺 (制比劫) > 食傷 (泄)
        return select_xi_shen_for_strong(chart)
    
    elif strength in ('偏弱', '弱極'):
        # 弱則扶之 / 助之 / 生之
        # ลำดับ:
        #   1. โดน 官殺 攻 → 用印 (化煞生身) > 比劫 (敵殺)
        #   2. โดน 食傷 泄 → 用印 (制食傷)

        return select_xi_shen_for_weak(chart)

        #          火金相戰 → 用土通關
        bridging = compute_bridging_element(chart.clashing_pair)

            'primary': bridging,
            'reason': f'{chart.clashing_pair} 相戰, 用{bridging}通其關'
        }
    
    # === STEP 5: 病藥用神 (refine the choice) ===
    # 《神峰通考·病藥說》: 「八字先觀月令何如，更看四柱有何病。有病方為貴，無病不是奇」
    bing = identify_disease(chart, candidate=current_yong_shen)

        # 藥 = สิ่งที่กำจัด/ลดทอน 病
        yao = find_cure(bing)
        return {
            'method': '病藥',
            'primary': yao,
            'reason': f'命局以{bing}為病, 取{yao}為藥'
        }
    
    return current_yong_shen

### 2.3 ลำดับความสำคัญเมื่อ Conflict (Decision Rules ละเอียด)

**กรณี 1: 扶抑 vs 調候 conflict**
- ตัวอย่าง: 丙火 生 子月 (เดือนหนาวมาก, มีน้ำมาก, 丙火อ่อน) — ตามตรรกะ扶抑ควรใช้印比 (木/火) เพื่อช่วย日主. ตามตรรกะ調候ก็เช่นกัน — **ในกรณีนี้ทั้งคู่ตรงกัน**
- ตัวอย่างที่ขัด: 庚金 生 午月 (ร้อนจัด, 火克金, 庚อ่อน) — 扶抑ต้องการ印比 (土/金) เพื่อช่วย日主, แต่調候ต้องการ壬癸水 (官殺) เพื่อดับร้อน — **rule**: เมื่อร้อน/หนาวสุดขั้ว, **調候ชนะ** (壬癸เป็นหลัก) แต่ก็ต้องดู根: ถ้าไม่มี印比สนับสนุนที่ฐาน, 壬癸ที่ใช้調候จะกลายเป็น killer; ในกรณีนั้นต้อง用印比เพื่อให้日主คงอยู่ก่อน (จาก《滴天髓·燥湿》任註: 「必須日主得以存活，方可尋局之寒暖濕燥」)

**กรณี 2: 病藥 จะ override 扶抑/調候 เมื่อ:**
- มี忌神 transparent ใน天干 ที่ทำร้าย用神ที่เลือกไว้ — ต้องลบ病ก่อน ไม่งั้น用神ใช้ไม่ได้
- ตัวอย่าง: 用神เป็น丙火, แต่天干透癸水 ใกล้ — 癸เป็น病, 用戊土合癸 (เป็น藥)

**กรณี 3: 從格 absolutely overrides 扶抑**
- 《滴天髓》: 「從得真者只論從」 — ถ้าเข้า從格จริง, ไม่ใช้扶抑เลย
- เกณฑ์เข้า從格 (เข้มงวด): (1) 日主無根 (no通根 in any branch) (2) 月令ไม่ช่วย (3) ไม่มี印比助 (4) one or more direction dominate
- **陽干**: 「陽干從氣不從勢」 — เข้า從ยาก, 丙火甲木เข้าได้ยากที่สุด
- **陰干**: 「陰干從勢無情義」 — เข้า從ง่าย, 乙木癸水เข้าง่ายที่สุด

#### 3.1.4 得勢 (Getting Momentum)

**คำนิยาม**: นับว่ามี干支ในดวงทั้งหมดที่生扶 (印星 + 比劫) เทียบกับที่剋洩 (財・官殺・食傷)

**Scoring System (สำหรับ engine implementation)**:

```python
def assess_strength(chart):
    score = 0
    
    # === 得令 (Month Command) — weight ~50% ===
    if month_branch_supports(day_master, month_branch):
        if state in ('帝旺', '臨官'):    score += 50
        elif state == '冠帶':           score += 35
        elif state == '長生':           score += 30
        elif state == '沐浴':           score += 10
        elif state in ('養', '胎'):     score += 5
        elif state == '衰':             score -= 5
        elif state in ('病','死'):      score -= 25
        elif state in ('墓','絕'):      score -= 30  # 陽干墓 +10 if has 墓根
    
    # === 得地 (Roots in branches) — weight ~30% ===
    for branch in [day_branch, hour_branch, year_branch]:
        if main_qi(branch) == same_element(day_master):
            score += 15      # 本氣根 = strong
        elif mid_qi(branch) == same_element(day_master):
            score += 8       # 中氣根
        elif residual_qi(branch) == same_element(day_master):
            score += 3       # 餘氣根
    
    # === 得勢 (Stems and support) — weight ~20% ===
    for stem in [year_stem, month_stem, hour_stem]:
        if stem_is_印星(stem, day_master):    score += 8
        elif stem_is_比劫(stem, day_master):  score += 10
        elif stem_is_食傷(stem, day_master):  score -= 6
        elif stem_is_財星(stem, day_master):  score -= 5
        elif stem_is_官殺(stem, day_master):  score -= 8
    
    # === Modifiers ===
    # 三合局 / 三會局 transforming branches
    if has_san_he(chart):
        if produces_印比(chart):  score += 20
        else:                      score -= 15
    if has_chong(chart, day_branch):  # 日支被沖
        score -= 10
    
    # === Classification ===
    if score >= 60:   return '旺極'      # → likely 從強/專旺
    elif score >= 30: return '偏旺'      # → 扶抑: 克泄耗
    elif score >= -15:return '中和'      # → 配合月令五行調適
    elif score >= -50:return '偏弱'      # → 扶抑: 生扶
    else:             return '弱極'      # → likely 從格
```

**กฎพิเศษ ('得時不旺，失時不弱') from 《滴天髓·衰旺》**:
- ถ้า得令แต่ทุก branches อื่นโจมตี日主, ยังถือว่า偏弱
- ถ้าไม่ได้令แต่มี3-4 branches รองรับ + 印星透干, ก็ถือว่า偏旺
- เป็นเหตุผลที่ต้องคำนวณ得令／得地／得勢ครบทั้ง 3 มิติ ไม่ใช่ดู月令อย่างเดียว

#### 3.1.5 Interaction & Final Determination

**Cascading rules**:
1. ถ้า得令 + 得地 + 得勢 ครบ 3 → **旺極 (太旺)** → 適合用克洩耗 (官殺/食傷/財)
2. ถ้า得令 + 得地 หรือ 得令 + 得勢 (ได้ 2) → **偏旺** → ใช้扶抑 (克洩耗)
3. ถ้า得令 อย่างเดียว — บางครั้งเป็น 中和 (ต้องดูสภาพ天干); 月令印比 vs 比劫ก็ต่างกัน
4. ถ้าได้แค่得地/得勢, 失令 → **偏弱** → ใช้扶抑 (印比生扶)
5. ถ้า 失令 + 失地 + 失勢 ทั้งหมด → **弱極** → ลองตรวจ從格

**สำคัญ**: ตามที่ระบุใน suanzhun.net BaZi 旺衰 article 「月令为比劫的力量，比月令是印星的力量更大」 — เพราะ印生身ผ่าน step, 比劫คือ日主โดยตรง

1. **Phase 1 — Static Reference Tables**: implement Section 1 (120-cell 調候 table) และ Section 3.1.3 (藏干 table) เป็น static lookup ก่อน. ใช้ JSON หรือ database table ที่ frozen + เก็บ version metadata (เช่น "余春台-光緒本", "徐樂吾-1937補註本", "梁湘潤-四明丁丑本")

## Caveats สำคัญ

1. **《窮通寶鑑》 มี variants** — ตามที่ระบุใน Baidu Baike: 「余春台，清朝光绪年间人士…光绪时有抄本在社会上流传，入楚南余春台之手，由其详加增删，细加编辑，并更名为《穷通宝鉴》刊行于世」 (Qing Guangxu period, 1875–1908). ต่อมา徐樂吾ทำ評註ใน 民國廿六年 (1937, ตาม進源書局 record). 「四明丁丑本」 variant และ尤偉武「造化玄鑰」 variant มีลำดับ用神แตกต่างกันบ้าง (เช่น 庚生寅月 บางบทเป็น 丙甲 บางบทเป็น 戊甲丙壬丁 รวม 5 ตัว). engine ควร store version metadata เพื่อให้ผู้ใช้เลือก
2. **「凡需調候者，一概暫緩論議」** — ประโยคนี้ถูกอ้างถึงในตำราสมัยใหม่ แต่จากการตรวจสอบเอกสารต้นฉบับครบทุกเล่มของ徐樂吾 (《子平真詮評註》、《窮通寶鑑評註》、《滴天髓補註》、《造化元钥評註》、《子平粹言》) **ไม่พบ verbatim quote** ทั้งสิบสามอักษรนี้; ที่ใกล้ที่สุดคือ徐樂吾《子平真詮評註·論用神配氣候得失》 (1936): 「用神須得時乘氣，譬如夏葛冬裘，得時則貴。然亦有用神雖乘旺氣而不貴者，則受氣候之影響。故取用神，於扶抑之外，必須參合氣候，即調候之法也」 — ใช้ phrase「凡需調候者，一概暫緩論議」อย่างระวัง, น่าจะเป็น secondary paraphrase
3. **得令 calibration** ระหว่าง 5-state กับ 12-stage — engine ควร support both; default ใช้ 12-stage เพราะแม่นกว่า
4. **月令分日 (司令)** มี traditions ที่ต่างกัน (เช่น巳月บางสำนักว่า戊7庚7丙16, บางสำนัก戊5庚9丙16) — เลือก default + ให้ admin override
5. **陰陽干墓庫** controversy: 陽干通墓 nearly all schools accept; 陰干通墓 schools แตก — engine ควรมี setting toggle
6. **從格 detection sensitive false positive** — recommend 2-tier: strict-from + loose-from, log both
7. **น้ำหนัก「月令 = 50%」 เป็น modern convention** ไม่ใช่ classical verbatim. ที่มาเดิมจาก《淵海子平》: 「月令者…生殺皆由之，故曰提綱」 และ《子平真詮》: 「八字用神，專求月令」 ไม่มีตัวเลข %; ตัวเลข 50% เป็นค่าที่นิยมในสำนัก旺衰派สมัยใหม่ (ตาม suanzhun.net และ similar BaZi portals). engine ควรให้ admin tune weight parameters
