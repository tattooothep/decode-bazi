### A2.0.2 ปรากฏการณ์การแก้ขัด (resolution phenomena)

- **貪合忘冲 (tan he wang chong)** — "เห็นแก่การรวม ลืมการชน": เมื่อกิ่งหนึ่งกำลังถูกชน แต่มีอีกกิ่งมารวมด้วย → การชนถูกระงับ. ที่มา《子平真詮·論刑沖會合解法》[verbatim] 「八字支中，刑沖俱非美事，而三會六合，可以解之。假如甲生酉月，逢卯則沖，而或支中有戌，則卯與戌合而不沖…是會合可以解沖也」
- **貪合忘剋 (tan he wang ke)** — เห็นแก่รวม ลืมควบคุม: ก้านที่ควรถูก 剋 โดยอีกก้าน ถูกอีกก้านมารวมแทน → หน้าที่ 剋 ดับ
- **貪生忘剋 (tan sheng wang ke)** — เห็นแก่ผลิต ลืมควบคุม: เช่น 庚 ควรจะ剋甲 แต่มี 壬 อยู่ตรงกลาง → 庚 เอนไปผลิต 壬 (金生水) แทนที่จะไปตี 甲
- **合化 (he hua)** — รวมแล้ว"แปร"เป็นธาตุใหม่: 甲己→土, 乙庚→金, 丙辛→水, 丁壬→木, 戊癸→火. ต้องมี 化神得令 ฯลฯ
- **合而不化 / 合絆 (he ban)** — รวมแต่ไม่แปร, เป็น"พันธนาการ"เท่านั้น: ธาตุยังอยู่ทั้งสอง แต่ทั้งคู่ "ถูกผูก" หน้าที่เดิมหยุดทำงาน
- **沖中逢合 (chong zhong feng he)** — กลางการชน เจอการรวม → 解沖 (ละลายการชน)
- **合處逢沖 (he chu feng chong)** — กลางการรวม เจอการชน → 沖開 (เปิดการรวม)

### A2.0.3 แยกชัด: 合化 vs 合絆

| มิติ | 合化 (true transform) | 合絆/合而不化 (bound only) |
|---|---|---|
| ธาตุ | เปลี่ยนเป็นธาตุใหม่ตามคู่ | คงเดิมทั้งสองธาตุ |
| หน้าที่สิบเทพ | คำนวณใหม่จาก化神 | "ถูกผูก" — ไม่ทำหน้าที่เดิม |
| เงื่อนไข | เข้มมาก (ดู A2.1) | เกิดได้ทันทีเมื่อสองก้านอยู่ติดกัน |
| ใช้กับ 用神 | ถ้า化神 เป็น喜 → เป็นบวกใหญ่ | ถ้า用神 ถูกผูก → 用神 พิการ (ลบใหญ่) |

**ผลต่อการอ่านจริง**: engine ต้องเช็ค 合化 ก่อน เพราะถ้าจริง ผังกลายเป็น 化氣格 (จัดอยู่ในไฟล์ A1) — ใช้ 用神/喜忌 แบบ化格. ถ้าไม่ใช่ 合化 ให้ตกลงไปที่ 合絆 ซึ่งกระทบ用神/喜忌 ผ่านการ "พิการชั่วคราว" ของก้านนั้น

## A2.1 天干五合 — เงื่อนไข 真化 vs 假化 vs 合而不化

### A2.1.1 ตารางคู่และ 化神

| คู่ก้าน | 化神 | เดือนที่化神得令 (จำเป็นต่อ真化) | สิ่งที่กลัวสุด (破化) |
|---|---|---|---|
| 甲己 | 土 | 辰戌丑未 (รองคือ午) | 木 (กระทบ化神土) |
| 乙庚 | 金 | 巳酉丑 (รองคือ申) | 火 (กระทบ化神金) |
| 丙辛 | 水 | 申子辰 (รองคือ亥) | 土 (กระทบ化神水) |
| 丁壬 | 木 | 亥卯未 (รองคือ寅) | 金 (กระทบ化神木) |
| 戊癸 | 火 | 寅午戌 (รองคือ巳) | 水 (กระทบ化神火) |

ที่มาเงื่อนไขเดือน:《三命通會·論十干化氣》(萬民英, 萬曆六年/1578) [verbatim] 「甲己化土，非辰戌丑未月不化，其次午月亦化，有戊字間之，則不化，名曰妒合」「乙庚化金，非巳酉丑月不化，其次七月亦化」「丙辛化水，非申子辰月不化，其次十月亦化」「丁壬化木，非亥卯未月不化，其次正月亦化」「戊癸化火，非丙午戌月不化，其次四月亦化」

### A2.1.2 真化 (true transformation) — เงื่อนไขเข้ม

ที่มาหลัก:《滴天髓·化象》[verbatim] 「化得真者只論化，化神還有幾般話」 พร้อม 原注 [verbatim] 「如甲日主生於四季，單遇一位己土，在月時上合之，不遇壬、癸、甲、乙、戊，而有一辰字，乃為化得真。又如丙辛生於冬月，戊癸生於夏月，乙庚生於秋月，丁壬生於春月，獨自相合，又得龍以運之，此為真化矣」

**เงื่อนไขที่ต้องครบทั้งหมด (engine: AND-gate)**:
1. 化神得月令 (化神อยู่ในเดือนที่ถูกต้องตาม A2.1.1)
2. 化神 透 ออกบนก้านสวรรค์ และมีรากในกิ่ง (透干 + 通根)
3. 日主无根 หรือมีรากอ่อนเท่านั้น (ถ้า日主มีรากแข็ง → ไม่ยอมไป化)
4. ไม่มี 爭合 / 妒合 — ไม่มีก้านอื่นที่จะรวมกับคู่ใดคู่หนึ่งได้ในเวลาเดียวกัน
5. ไม่มีธาตุ剋化神อย่างมีอำนาจในผัง
6. (เฉพาะ甲己) มี 辰字 (มังกร) ในกิ่ง — เพราะ "得龍以運之"

หลังจริงแล้ว: [verbatim《滴天髓·化象》原注] 「既化矣…再見甲乙，亦不作爭合妒合論。蓋真化矣，如烈女不更二夫，歲運遇之皆閒神也」 — เมื่อ真化แล้ว การเจอคู่อื่นๆในภายหลัง ไม่ถือว่า 妒合 อีก

**ผลต่อการอ่านจริง**: 真化 ทำให้ผังกลายเป็น 化氣格. 用神 = สิ่งที่หล่อเลี้ยง 化神 + สิ่งที่ระบาย化神 (ดูหลักประกอบใน A1). 忌神 = สิ่งที่ 剋 化神

### A2.1.3 假化 (false transformation)

ที่มา:《滴天髓·假化》[verbatim] 「假化之人亦多貴，孤兒異姓能出類」 พร้อม 原注 [verbatim] 「日主孤弱而遇合神真，不能不化，但暗扶日主，合神又虛弱，及無龍以運之，則不真化。至於歲運扶起合神，制伏忌神，雖為假化，亦可取富貴，雖是異姓孤兒，可出類拔萃。但其人多執滯偏拗，作事迍邅，骨肉欠遂」

**เงื่อนไข 假化 (อย่างน้อยหนึ่งข้อ ที่ทำให้真化ไม่ครบ แต่ยังมีแนวโน้มไปทาง化):**
- 日主 อ่อนสุด ไม่มีทางพึ่ง → "不能不化" (ฝืนต้องไป)
- 化神 ไม่ได้เดือน หรือไม่มี辰 (กรณี甲己)
- มี閒神มาก่อกวนเล็กน้อย แต่ไม่ทำลายโดยตรง

**ผลต่อการอ่านจริง**: 假化 ถือเป็น化氣格ที่"ไม่บริสุทธิ์". 用神 ยังเป็นการเสริม化神 แต่ engine ต้อง mark `purity_flag = false` และต้องระวัง 歲運 ที่ปลุก日主 (ฟื้นรากเดิม) → ผังพังทันที. 任鐵樵 ขยายในความเห็นเฉพาะของตน [paraphrase《滴天髓闡微·假化》任氏曰] ว่า "假化比真化尤難，更宜細究"

### A2.1.4 合而不化 / 爭合 / 妒合 (合絆 only — no transform)

ที่มา:《子平真詮·論十干合而不合》ของ 沈孝瞻

- **กรณี A: 隔位太遠 (距離ไกล)** [verbatim] 「又有隔位太遠，如甲在年干，己在時上，心雖相契，地則相遠，如人天南地北，不能相合一般。然於有所制而不敢合者，亦稍有差，合而不能合也，半合也，其爲禍福，得十之二三而已」 → 沈 มอง "遙合" เป็น "ครึ่งหนึ่งของการรวม" — มีแต่ลด. **ตามอักษรเดิมคือ "得十之二三" คือออกฤทธิ์เพียง "สอง-ถึง-สามในสิบส่วน"** (ตัวเลขนี้เป็น internal weight เท่านั้น ห้ามแสดงต่อลูกค้า — ดูภาคผนวก)
- **กรณี B: 中間有隔 (มีก้านอื่นกั้น)** [verbatim] 「假如甲與己合，而甲己中間以庚間隔之，則甲豈能越尅我之庚而合己？此制於勢者然也，合而不敢合也，有若無也」
- **กรณี C: 爭合/妒合 (competing combination)** [paraphrase อิง《三命通會·論十干化氣》] เช่นมี甲สองตัวรวมกับ己ตัวเดียว, หรือ乙สองตัวรวมกับ庚ตัวเดียว → "一女二夫" สองคนแย่งกัน → ไม่มีฝ่ายใดได้ครอบครอง → คงสภาพ"พันธนาการอย่างเดียว" (合絆) ไม่แปร
- **กรณี D: 鄰位但日主根強 (อยู่ติด แต่日主มีราก)** → ก้านไหนเป็น日干จะปฏิเสธไป化 → คง 合絆 ก้านอีกฝั่งหน้าที่หยุดทำงาน
- **กรณี E: 同柱自合 (อยู่บนเสาเดียวกัน)** เช่น 戊子 (戊กับ子中癸), 辛巳, 丁亥, 壬午 → 任鐵樵 [verbatim《滴天髓闡微·配合章》任氏曰] 「然可言合者，只有戊子、辛巳、丁亥、壬午四日耳」 — เฉพาะสี่วันนี้เท่านั้นที่ถือเป็น"天合地"ที่นับเป็น 合 ได้จริง

**ผลต่อการอ่านจริง**:
- ถ้า用神ถูก 合絆 → 用神พิการชั่วคราว/ถาวร → reading พลังตก (合住用神)
- ถ้า忌神ถูก 合絆 → 忌神ถูกควบคุม → reading ดีขึ้น (合去忌神)
- engine ต้อง trace ว่าก้านไหนกำลังพันธนาการก้านไหน ก่อนสรุป 用神 reading

### A2.1.5 ตารางวิศวกรรม — ก้านสวรรค์ 5 คู่ × 3 outcome

| คู่ | 化神 | 真化 strict (ต้องครบทั้งหมด) | 假化 (กึ่ง) | 合而不化 (合絆) | ผลต่อ 用神 |
|---|---|---|---|---|---|
| 甲己 | 土 | 日主อ่อน + 月令 辰戌丑未 + 透土通根 + 有辰字 + ไม่มี乙/木剋 + ไม่爭合 | ขาด辰 หรือ月令รอง(午) + 日主弱สุด | 隔位/中間有水木/日主有根/มี甲คู่ที่สอง | 化:用土系; 絆:หาก甲เป็น用神→พิการ |
| 乙庚 | 金 | 月令 巳酉丑 + 透金通根 + 日主弱 + ไม่มี丙丁剋金 + ไม่爭合 | 月令 申 + 日主弱 | 隔位/มี火โต/มี乙คู่ที่สอง/มี庚คู่ที่สอง | 化:用金系; 絆:乙/庚ที่เป็น用神พิการ |
| 丙辛 | 水 | 月令 申子辰 + 透水通根 + 日主弱 + ไม่มี戊己土มาขุ่น + ไม่爭合 | 月令 亥 + 日主弱 | 隔位/มี土แรง/มี丙หรือ辛คู่ที่สอง | 化:用水系; 絆: |
| 丁壬 | 木 | 月令 亥卯未 + 透木通根 + 日主弱 + ไม่มี庚辛剋木 + ไม่爭合 | 月令 寅 + 日主弱 | 隔位/มี金แรง/มี丁หรือ壬คู่ที่สอง | 化:用木系; 絆: |
| 戊癸 | 火 | 月令 寅午戌 + 透火通根 + 日主弱 + ไม่มี壬水มา剋 + ไม่爭合 | 月令 巳 + 日主弱 | 隔位/มี水แรง/มี戊หรือ癸คู่ที่สอง | 化:用火系; 絆: |

**Engine flow**: เรียงลำดับเช็ค → (1) 真化 ครบไหม → ถ้าใช่ ไป化氣格 (2) ถ้าไม่ครบแต่日主弱สุด → 假化 + flag purity false (3) นอกนั้น → 合絆 + คำนวณว่าก้านใดพิการ และก้านนั้นเป็น 用/喜/閒/忌 หรือไม่

## A2.2 地支 — บัญชีปฏิสัมพันธ์ทั้งหมด

### A2.2.1 地支六合 (six-combinations)

| คู่ | สิ่งที่แปรเป็น (เงื่อนไข) | ความสัมพันธ์ภายในคู่ |
|---|---|---|
| 子丑 | 土 (ถ้ามีรากดิน) | 土剋水 (剋合) |
| 寅亥 | 木 (เงื่อนไขเบา) | 水生木 (生合) |
| 卯戌 | 火 (เงื่อนไขเข้ม — เพราะ卯ไม่ใช่ฐานไฟ) | 木剋土 (剋合) |
| 辰酉 | 金 | 土生金 (生合) |
| 巳申 | 水 (เงื่อนไขเข้ม) | 火剋金/金生水 (พลังขัดในตัว — มักจะแค่絆) |
| 午未 | 火/土 (ขึ้นกับว่าทิศร้อน) | 火生土 (生合) |

หลักทั่วไป (徐樂吾《評註》[paraphrase] + 滴天髓 ปฏิบัติ): 六合ส่วนใหญ่ใน四柱 = 合絆 ไม่ใช่ 合化 เว้นจะมี透干+月令ของ化神

**ผลต่อการอ่านจริง**: engine ตั้งค่า default ของ 六合 = 合絆 (ผูกแต่ไม่แปร); 化 เฉพาะเมื่อมีหลักฐานเสริมพอ (化神透干, 月令ส่งเสริม)

### A2.2.2 三合局 (three-harmony bureau)

| ฐาน局 | 長生 | 帝旺 (中神) | 墓 | ครบสามตำแหน่ง = 全局 |
|---|---|---|---|---|
| 水局 | 申 | 子 | 辰 | 申子辰 |
| 木局 | 亥 | 卯 | 未 | 亥卯未 |
| 火局 | 寅 | 午 | 戌 | 寅午戌 |
| 金局 | 巳 | 酉 | 丑 | 巳酉丑 |

**Variants**:
- **半合** (half-combination): มี 帝旺 + 長生 (เช่น 申子) หรือ 帝旺 + 墓 (เช่น 子辰) — ยอมรับเป็น局 แต่อ่อนกว่า全
- **拱合** (embracing): กิ่งสองตำแหน่งของ三合 ขนาบกิ่งกลาง (กลางหายไป) แต่ก้านบนเสาแทน → ยังนับเป็น局ครึ่ง. เกณฑ์ที่ 徐樂吾 บันทึก [verbatim《子平真詮評註》第07章] 「若支寅戌而干丙丁，支申辰而干壬癸，則仍可成局，丙丁即午，壬癸即子也」
- **無中神** (เช่น寅戌, 申辰): "ไม่เป็น局" เป็นเพียงสองกิ่งเฉยๆ [verbatim《子平真詮評註》第07章] 「若單是寅戌或申辰，則不成局」 แต่ "又寅戌會無午而有巳，申辰會無子而有亥，亦有會合之意" — มี 祿 ของก้านกลางก็พอใช้เป็น"กึ่งครบ"

**化神ต้อง透干ไหม?**: ขัดแย้งระหว่างสำนัก — ดู A2.5 ข้อที่ 4

### A2.2.3 三會方 (directional meeting)

| ทิศ | สามกิ่ง | ธาตุ |
|---|---|---|
| ตะวันออก | 寅卯辰 | 木 |
| ใต้ | 巳午未 | 火 |
| ตะวันตก | 申酉戌 | 金 |
| เหนือ | 亥子丑 | 水 |

**สถานะตำราหลัก** —《子平真詮·論刑沖會合解法》[verbatim] บันทึกว่า 沈孝瞻 ใช้คำว่า "三會" สำหรับสิ่งที่ตำราอื่นเรียก 三合 (เช่นเรียก申子辰ว่า "三會"). บรรณาธิการ 東里山人 (ของฉบับ donglishuzhai.net ปี 2012) ใส่หมายเหตุชัด [verbatim] 「他書『三會』乃一氣相連，如寅卯辰之類。若申子辰者，乃三合也…沈氏實以三合爲『三會』，而本書實未有他書所稱之『三會』者」 → ความขัดแย้งทางคำศัพท์นี้ engine ต้องตระหนัก: ใช้คำนิยามมาตรฐาน (寅卯辰=三會方, 申子辰=三合局)

**ลำดับความแรง (default, ดู A2.5)**: 三會方 > 三合局全 > 三合半 > 六合

**ผลต่อการอ่านจริง**: ถ้ามี三會方ครบ engine ควรพิจารณาว่าผังเข้าข่าย "專旺/方局格" (อยู่ใน A1) หรือไม่ — ถ้าใช่ 用神 จะอ่านตามแบบ 專旺格 (順其勢)

| 卯酉 | สี่ทิศ — ชนกลาง |
| 寅申 | 生地 (生方) — 任鐵樵 [verbatim《滴天髓·地支》] 「生方怕動者，兩敗俱傷也…戰剋不靜故也」 |

### A2.2.5 三刑/自刑

## A2.2.1★ DECISION TREE — ลำดับความสำคัญหลัก (Authoritative resolution hierarchy)

> ส่วนนี้คือ **"คำตอบเดียว"** ที่ engine ใช้เมื่อเกิดปฏิสัมพันธ์ทับซ้อนบนกิ่งเดียวกัน. ลำดับนี้คือ default ของระบบ ส่วนการพลิกลำดับให้ดู A2.2.2 conditional layer

### ลำดับการตรวจ (top-down; engine runs in order)

```
STEP 0  ตรวจ反吟/伏吟ระดับ柱 (pillar-level repeat/mirror) — flag เก็บไว้ แต่ยังไม่ resolve
STEP 1  ตรวจ天干合 ทั้งหมด → ตัดสิน 真化 / 假化 / 合絆
        - ถ้า 真化 → ผังกลายเป็น化氣格 (ออกจากตรรกะปกติ)
        - ถ้า 合絆 → mark ก้านที่ถูกผูก (พิการ)
STEP 2  ตรวจ地支ระดับ會/合/沖/刑/害/破 ในลำดับความแรง:
        2a  三會方 ครบ      (แรงสุด)
        2b  三合局全 (3 ตัว)
        2c  三合半局 + 拱合
        2d  地支六合
        2e  地支六沖
        2f  三刑/自刑       (low — ดู conditional)
        2g  六害/穿         (lowest)
        2h  破               (ตัดทิ้ง default)
STEP 3  ใช้กฎ"貪合忘冲" — สำหรับทุกคู่ที่มีทั้ง合ทั้ง沖บนกิ่งเดียวกัน
STEP 4  ใช้กฎ"合處逢沖 / 沖中逢合"
STEP 5  Map ผลลัพธ์เข้ากับ 用神/喜忌 (binding/release)
```

### A2.2.1.1 หัวข้อขัดแย้งคลาสสิก — คำตอบ default

**Q1: 合 ละลาย 冲 ไหม (貪合忘冲)?**
> **A1 (default)**: ใช่ — ถ้าก้านที่มารวมอยู่ติดกับกิ่งที่ถูกชน. ที่มา:《子平真詮·論刑沖會合解法》[verbatim] 「八字支中，刑沖俱非美事，而三會六合，可以解之。假如甲生酉月，逢卯則沖，而或支中有戌，則卯與戌合而不沖…是會合可以解沖也」 — 沈孝瞻ยอมรับเต็ม.《滴天髓·合局》原注 [verbatim] 「動局有能合而靜者，如子午相沖，得丑合而靜」
> **ข้อยกเว้น**: ถ้า冲เป็นการ"ดี" (เช่นมาเปิดคลัง庫 หรือชนล้าง忌神 — เลือกใช้ 沖) → ไม่ละลาย. 任鐵樵 [verbatim《滴天髓·合局》任氏曰] 「若忌合而合之，比沖愈凶也」 — เตือนว่า"การรวมที่ไม่ดี"จะร้ายยิ่งกว่าการชน. ดู conditional A2.2.2

**Q2: 三會 > 三合 > 六合 จริงไหม?**
> **A2 (default)**: ใช่. 三會 = "一氣相連" หนึ่งฤดูเต็ม. 三合 = "三方為朋友". 六合 = "並對為鄰". ลำดับนี้สอดคล้องกับ任氏ในการใช้ตัวอย่าง.《子平真詮·論刑沖會合解法》[verbatim] 「三方為會，朋友之意也；並對為合，比鄰之意也」 — 沈ใช้ "會" สำหรับสิ่งที่ผู้อื่นเรียก "三合" → engine ใช้คำนิยามใหม่ ไม่ใช่ของสำนัก沈; แต่ลำดับ แรง→อ่อน ยังคงเดิม

**Q3: 刑/害 อยู่ตรงไหนเทียบกับ 合/沖?**
> **A3 (default)**: 沖 > 合 > 刑 > 害 > 破. รากที่มา: 任鐵樵 [verbatim] 「支神只以沖為重，刑與穿兮動不動」. การ 刑/害 มีผลก็ต่อเมื่อก้านที่ถูกกระทบเป็น用神หรือ喜神อยู่แล้ว — ไม่มีอำนาจ"แตก"การรวมที่ดีหรือ"กลบ"การชนแรง

**Q4: 沖 แตก 三合/六合 ได้ไหม (沖開)?**
> **A4 (default แบบ"แยกกรณี")**:
> - 沖ที่กระทบ "ก้านกลาง帝旺" (子/午/卯/酉) ของ三合: **แตก全局** (สำคัญสุด)
> - 沖ที่กระทบ "ก้านปลาย" (長生หรือ墓) ของ三合: **ลดแรง** ไม่แตก
> - 沖ที่กระทบหนึ่งในคู่六合: **แตก合** (เพราะ六合มีพันธบาง)
> - 三會ครบ: 沖ก้านเดียวแทบไม่กระทบ (มวลฤดูใหญ่กว่า)
>
> สอดคล้อง 任氏曰 [verbatim《滴天髓·合局》] 「沖得合而靜者則易，合得沖而開者則難」 — กล่าวคือ"合แก้沖ง่าย, 沖แตก合ยากกว่า" — แต่ก็ทำได้

**Q5: 天干 vs 地支 — แยกหรือสัมพันธ์?**
> **A5 (default)**: คำนวณแยก แต่ผลกระทบสองชั้น
> - ชั้น天干: คำนวณ五合 + 剋 → กระทบ"การปรากฏ"ของสิบเทพในก้าน
> - ชั้น地支: คำนวณ會/合/沖/刑/害 → กระทบ"พลังราก" และคลัง藏干
> - **เชื่อม**: 透干ของ会化ในกิ่ง (เช่น 寅午戌 + 丙ก้าน → ยืนยัน化火) → "天合地" (เช่น 甲子+己丑) ที่ 任鐵樵 ระบุ [verbatim《滴天髓·天合地》原注] 「如丁亥、戊子、甲午、己亥、辛巳、壬午、癸巳之類，皆支中人元，與天干相合者」

### A2.2.1.2 Flowchart ตรรกะ (engine-runnable pseudocode)

```
function resolve(pillars):
  results = []
  // STEP 1 — Heavenly stems
  for each stem_pair in find_5he(pillars):
    if is_true_transform(stem_pair, month):
      results.add( STEM_HUA(化神) )
    elif is_false_transform(stem_pair):
      results.add( STEM_HUA_FALSE(化神, purity=false) )
    else:
      results.add( STEM_BAN(stem_pair) )  // 合絆

  // STEP 2 — Earthly branches by strength
  if has_san_hui(pillars):     results.add( BRANCH_SAN_HUI )    // ทิศเต็ม
  if has_san_he_full(pillars): results.add( BRANCH_SAN_HE_FULL )
  if has_san_he_half(pillars): results.add( BRANCH_SAN_HE_HALF )
  if has_liu_he(pillars):      results.add( BRANCH_LIU_HE )
  if has_liu_chong(pillars):   results.add( BRANCH_CHONG )
  if has_xing(pillars):        results.add( BRANCH_XING )
  if has_hai(pillars):         results.add( BRANCH_HAI )

  // STEP 3 — 貪合忘冲 resolution
  for each chong in results.chongs:
    if exists_adjacent_he_on_same_branch(chong):
      mark chong as RESOLVED_BY_HE
      mark both branches as WEAKENED (but not destroyed)

  // STEP 4 — 沖開 (clash opens combination)
  for each he/sanhe in results.combinations:
    if exists_chong_on_center_branch(he):
      mark combination as BROKEN
    elif exists_chong_on_edge_branch(sanhe):
      mark combination as REDUCED

  // STEP 5 — Map to 用神/喜忌
  for each used_god in chart.yongshen:
    if used_god is bound: flag REDUCED_FUNCTION
    if used_god is freed: flag RESTORED_FUNCTION
  return final_state
```

## A2.2.2★ CONDITIONAL LAYER — เงื่อนไขที่ "พลิก" ลำดับใน A2.2.1

> footnote-style conditional modifiers ที่วางทับ decision tree. ทุกข้อมีรูปแบบ "WHEN <condition> THEN <flip>"

### A2.2.2.1 位置 (position) — เสาไหนเกี่ยวข้อง

- **月支 (เดือน) เป็น 提綱**: ปฏิสัมพันธ์ใดๆที่กระทบ月支 มีน้ำหนักสูงสุด เพราะ月支คือฐานของ用神และ格局 (อ้างอิงตรงกับ A1 — ทุก格จาก月令)
  - WHEN 用神อยู่ใน月支 AND ถูก合住 THEN ผังพังสนิท
  - WHEN ถูก冲โดย年支/時支 THEN แล้วแต่ว่าสิ่งที่ถูกชนเป็น用หรือไม่
- **日支 (วัน) = 用神座/夫妻宮**: ถ้าถูก合住 → 日主 "ถูกผูก" → กระทบบุคลิก+คู่ครอง
- **時支 (เวลา) = 子女宮 + 歸宿**: 冲時支ปลายชีวิตหรือลูกหลานสะเทือน
- **年支 (ปี) = 祖宮**: ระยะไกลจาก日主, 冲ที่นี่กระทบน้อยถ้า月日มั่นคง

**Engine rule**: weight order ของกิ่ง (เก็บใน internal params, ดูภาคผนวก) — แต่หลักคือ "月支 > 日支 > 時支 > 年支" สำหรับเรื่อง用神; "日支" สำหรับเรื่องคู่ครอง

### A2.2.2.2 距離 (distance) — 近 vs 隔位

ที่มาขัดแย้ง:
- 沈孝瞻 [verbatim《子平真詮·論十干合而不合》] 「隔位太遠…合而不能合也，半合也，其爲禍福，得十之二三而已」 → ลดลงเหลือ "十之二三" (สอง-ถึง-สามในสิบ — internal weight; ห้ามแสดงต่อลูกค้า)
- 徐樂吾 [verbatim《子平真詮評註》第05章] 「隔位太遠，則合之效用減少，有以失其原來之力為喜。有以不失其力為喜。或雖遙隔而仍作合論，各視其格局配合而已」 → ไม่ตัด แต่"แล้วแต่กรณี"
- 任鐵樵《滴天髓》: ไม่ได้ตั้งกฎตายตัว แต่ผ่านตัวอย่างหลายชิ้นมัก "ยอมรับ 遙合" หาก化神有勢

**Engine default (พลิกตามเงื่อนไข)**:
- WHEN 兩干隔ก้านอื่นที่"ขวาง剋ก้านใดก้านหนึ่ง" THEN ไม่合 (เพราะถูก"ปกป้อง")
- WHEN 隔แค่ก้านที่"ไม่ขวาง" THEN ลดแรง (engine ใช้ flag `DIST_FAR`)
- 地支同樣: WHEN 三合ของกิ่งที่ห่างเสาไม่ติด THEN ลดแรง แต่ยังเป็น半局
- **隔位之合力減, 隔位之冲亦減** (สมมาตร) — ทั้ง合และ冲ลดเท่ากันเมื่อห่าง

### A2.2.2.3 數量 (count) — จำนวนปฏิสัมพันธ์บนกิ่งเดียวกัน

| สถานการณ์ | คำตอบ default |
|---|---|
| 一冲一合 (1ชน + 1รวม) | 貪合忘冲 (รวมชนะ) แต่ทั้งคู่อ่อนลง |
| 兩合一冲 (2รวม + 1ชน) | รวมชนะแน่ — เพราะมี"พันธ"สองแรงยัด |
| 兩冲一合 (2ชน + 1รวม) | ชนชนะ — รวมไม่แข็งพอ ตามที่《子平真詮·論刑沖會合解法》[verbatim] 「假如甲生子月，支逢二卯相並，二卯不刑一子」 — ใช้กับ 刑 ด้วยตรรกะเดียวกัน |
| 一合二爭 (妒合/爭合) | ทั้งคู่合絆ไม่แปร — "เลือกใครไม่ได้" |
| 三合 + 沖ก้านกลาง | 沖開 — 三合แตก |
| 三合 + 沖ก้านขอบ | 三合ลดแรง — ไม่แตก |

หลักประกอบ:《子平真詮·論刑沖會合解法》[verbatim] 「又有因解而反得刑沖者…二卯不刑一子，而支又逢戌，戌與卯合，本爲解刑，而合去其一，則一合而一刑」 → engine ต้องจัดการ"การลดจำนวน"และ"การเปิดทาง"ให้ครบ

### A2.2.2.4 動靜 (moving vs static) — 大運/流年 vs 原局

- **原局 (natal)** = static interaction: เกิดตลอดชีวิต
- **大運 (10-yr luck)** + **流年 (annual)** = moving: เกิดเฉพาะช่วง
- **歲運合原局** เรียก "合動" (เกิดการเคลื่อน): ปลุก/ผูก/ปลด ปฏิสัมพันธ์ในผังต้นกำเนิด
- **歲運沖原局**: หากกิ่งเดิมเป็น用神 → ระวังภาวะ"พังหรือเปิด"
- **引動** (triggering): เช่น原局มี 寅戌 (拱火 half) แต่ไม่มี午 → ถ้า流年เป็น午 → ปลุกครบ三合火局

**ที่มา**:《三命通會·總論歲運》(萬民英 1578) [verbatim] 「行運以生月為運元，最怕行運與太歲冲剋，若歲運冲月必禍。若歲運與日相對，謂之返吟。歲運壓日，謂之伏吟。二者不利六親，非橫破財，不為吉兆」

**Engine rule**:
- ปฏิสัมพันธ์ใน原局ติดถาวร (high weight ในภาคผนวก)
- ปฏิสัมพันธ์ที่มา"จาก運" — flag `DYNAMIC_LUCK`, ปลด/ผูกเฉพาะช่วง
- เมื่อ運 introduce กิ่งใหม่ที่เติม半局→全局 → flag `TRIGGER_EVENT` (เหตุการณ์ปลุก)

| (interaction, position, dist, count, src) | Outcome |
|---|---|
| (沖, 月日相鄰, 近, 1, 原局) | 重沖 — ผังสะเทือน, ถ้าเป็น 月支用神 → ต้องเช็ค救應 |
| (沖, 年時, 遠, 1, 原局) | 輕沖 — กระทบเฉพาะ6親 |
| (沖, 月日相鄰, 近, 1, 原局) + (合, 月日相鄰, 近, 原局) | 貪合忘冲 — ชนถูกแก้ |
| (三合全, _, _, _, 原局) + (沖中神) | 沖開 — 三合แตก |

| (天干合, 隔ก้านขวาง剋, _, _, _) | 不合 — ตามที่ 沈孝瞻 [verbatim《子平真詮》ch.5] |

| (伏吟柱, _, _, _, 流年=日柱重) | "轉趾煞" — แลกแกน [verbatim《三命通會·看命口訣》] 「日時干支與流年干支同，為轉趾煞」 |

## A2.3 沖中逢合 / 合處逢沖 — กรณีเฉพาะ

### A2.3.1 沖中逢合 (clash, but combination intervenes)

**คำนิยาม**: บนกิ่งที่กำลังถูก冲 มีอีกกิ่งมา合กับฝั่งใดฝั่งหนึ่ง → 合 "ระงับ" การชน

**ที่มา**:《子平真詮·論刑沖會合解法》[verbatim] 「假如甲生酉月，逢卯則沖，而或支中有戌，則卯與戌合而不沖；有辰，則酉與辰合而不沖；有亥與未，則卯與亥未會而不沖；有巳與丑，則酉與巳丑會而不沖。是會合可以解沖也」

**เงื่อนไขที่ engine ยอมรับ**:
- ก้านที่มาแก้สอบ "อยู่ติด" กับกิ่งที่ถูกชน (近)
- ก้านที่มาแก้แข็งพอ (มีรากในเดือนของตัวเอง)
- ก้านที่มาแก้ "ไม่ใช่忌神" (ถ้าเป็น忌神 ก็แก้ได้แต่จะนำผลเสียมาแทน)

**ข้อยกเว้น**: ถ้า冲ที่เกิดเป็น "冲ที่ดี" (เช่นเปิดคลังเก็บ用神, ตี忌神ออก) → engine ไม่ควรให้合มา"ปิด"การชนที่เป็นประโยชน์

**ผลต่อการอ่านจริง**: 沖ที่จะทำให้ 用神/喜神 พังถูก合มาช่วย → 用神 รอด → reading ดี; แต่ถ้าก้านที่มาช่วยเป็น忌神 → "เอาหนามถอนหนาม" ผลข้างเคียงเป็นการพิการ用神อีกทาง

### A2.3.2 合處逢沖 (combination, but clash intervenes — 沖開)

**คำนิยาม**: 合 (สอง/สาม) กำลังเกิด แต่มีกิ่งที่เข้ามาชนทำลาย → 合 แตก

**ที่มา**:《子平真詮·論刑沖會合解法》[verbatim] 「又有刑沖而會合不能解者，何也？假如子年午月，日坐丑位，丑與子合，可以解沖，而時逢巳酉，則丑與巳酉會，而子復沖午」

**ลำดับ engine**:
1. ตรวจ合ก่อน
2. ตรวจว่า"กิ่งที่อยู่ในการรวม"มีถูกชนไหม
3. ถ้า中神 (帝旺) ของ三合ถูกชน → 三合แตก
4. ถ้ากิ่งหนึ่งของ六合ถูกชน → 六合แตก (เพราะ六合มีพันธอ่อน)

### A2.3.3 貪合忘冲 — กลไกรายละเอียด

**ตัวอย่างสำคัญที่ 徐樂吾 (Xu Lewo, 1886–1948) ยกใน《子平真詮評註》第07章** [verbatim, 1936 edition]:
- 「壬午 壬子 庚辰 甲申」 — เสากลาง子午冲, แต่มี 申子辰會水 → "因申子辰之會，而解子午之冲也"
- 「戊午 辛酉 乙卯 丙戌」 — มี 卯酉冲, แต่มี 卯戌合 → "因卯戌之合，而解卯酉之冲也"
- 「甲子 丙子 丙寅 丙申」 — 寅申冲, แต่มี子申會 → "因寅申之冲，而解子申之會也" (กลับด้าน — สั้นๆ:沖มาทำลาย會)

**ข้อสำคัญที่ engine ต้องระวัง**: 徐註 [verbatim] 「會合可以解刑冲，刑冲亦可以解會合。此須看地位與性質之如何而定，有冲之無力，冲如不冲者，法至活變，無一定之方式也」 — ไม่มีกฎตายตัว 100% engine ต้องเช็คเป็นรายเคส

### A2.3.4 三合/六合 被沖 — แตกทั้งหมดหรือแค่กิ่งที่ถูก?

**Default ที่ engine ใช้**:

| สถานการณ์ | ผล |
|---|---|
| 三合全 (3 กิ่ง) + 沖ที่กิ่งกลาง 帝旺 | **แตก**ทั้ง局 — เพราะ帝旺เป็นหัวใจ |
| 三合全 + 沖ที่กิ่งปลาย (長生 หรือ墓) | **ลดแรง** — 局ยังอยู่แต่อ่อน |
| 三合半 + 沖กิ่งใดก็ตาม | แตก半局 (เพราะรากบาง) |
| 六合 + 沖กิ่งใดก็ตาม | แตก六合 |
| 三會方ครบ + 沖 | แทบไม่กระทบ (ฤดูแรงเหนือเรื่องเล็ก) |

**ที่มา synthesized**: จาก 徐註《子平真詮》ที่ว่า "三合以四正為主" (帝旺เป็นแกน) + 任鐵樵 ที่ "支神只以沖為重" + ปฏิบัติของ任ในการชี้ว่า沖กลาง局มักทำลาย

**任氏ยังขยาย** [paraphrase《滴天髓·地支》任氏曰] ว่า 「冲旺神」 เป็นเรื่องร้ายหาก "忌神冲旺神" แม้แตกก็ก่อโรค; แต่ "冲閒神" หรือ "冲忌神" หากเป็น"ทิศทางดี"ก็ปลด喜ออกได้

### A2.3.5 反吟 (fan yin) / 伏吟 (fu yin) — กรณีพิเศษ柱-ระดับ

**คำนิยาม**:
- **伏吟 (fu yin)**: 干支ซ้ำ (เช่น年柱甲子, 流年甲子, หรือ日柱壬寅 + 月柱壬寅)
- **反吟 (fan yin / 返吟)**: 干剋干 + 支冲支ในคู่เดียวกัน (เช่น年柱甲子, 流年庚午 — 庚剋甲 + 子冲午 = 天剋地沖)

**ที่มา**:《三命通會·總論歲運》[verbatim] 「歲運與日相對，謂之返吟。歲運壓日，謂之伏吟。二者不利六親，非橫破財，不為吉兆」
《三命通會·看命口訣》[verbatim] 「日時干支與流年干支同，為轉趾煞。如庚申日見庚申或庚寅太歲之類，輕則遠遷，重則毀屋破財」

**Engine weighting (default)**:
- 反吟柱ภายในผังกำเนิด (เช่น 月柱與日柱反吟): high weight permanent — กระทบ6親ในเสาที่เกี่ยวข้องตลอดชีวิต
- 反吟ระหว่าง大運/流年กับเสาเดิม: high weight ช่วงนั้นเท่านั้น — มัก triggering event
- 反吟柱 + 子午冲ภายในผัง: น้ำหนัก"ซ้อน" — engine ต้องคำนวณทั้งระดับเสาและระดับกิ่งแยกกัน แล้วรวมผล
- 伏吟柱: weight ต่ำกว่า反吟ในแง่ damage แต่ flag "repetition" — กระทบบุคลิก/ที่อยู่/พ่อแม่กับลูก

**ผลต่อการอ่านจริง**: เมื่อมี反吟คู่ (เช่น 子午冲 + 全柱反吟) — engine ต้อง:
1. ระดับกิ่ง: คำนวณ子午冲ผ่าน A2.2.1 → ถ้าเป็น用神被冲 → reading พลังตก
2. ระดับเสา: เพิ่ม pillar-level weight (กระทบบุคลิก/6親ของเสาตามตำแหน่ง)
3. ถ้าเป็น反吟ใน大運/流年: flag "事件年" — events มักเป็น 換工作/搬家/分離 ตามที่《三命通會》ระบุ ("非橫破財，不為吉兆")

## A2.4 合 ผูก 用神 หรือ 忌神 — ผลปฏิบัติ

### A2.4.1 合住用神 (binding the useful god — ผลลบ)

**คำนิยาม**: 用神 ถูกอีกก้าน/กิ่งมารวมจน "พันธนาการ" (合絆) → 用神 ไม่ทำงาน ผังพิการ

**ตัวอย่างคลาสสิก**:《子平真詮·論十干合而不合》[verbatim] 「如甲用辛官，年丙月辛，丙與辛合，是爲合官」 — ใช้辛เป็น正官 แต่ปี丙มารวม辛 → 官星ถูกผูก → ใช้ไม่ได้ → ผังพิการ

**Engine handling**:
- WHEN 用神ถูก合住 AND ไม่มีก้าน/กิ่งเดียวกันที่ replicate ได้ THEN flag `YONG_SHEN_BOUND`
- WHEN 用神ถูก合住 AND มี救應 (เช่น มี忌神มาตี/合คู่ที่บล็อก) THEN downgrade เป็น "ลดประสิทธิภาพ"
- WHEN 用神 อยู่ใน月支 AND ถูก合จาก日支/時支 THEN ผลรุนแรงสุด

### A2.4.2 合去忌神 (combining away the unfavorable god — ผลบวก)

**คำนิยาม**: 忌神 ถูกอีกธาตุมารวมหรือนำธาตุไป → 忌神 ถูกควบคุม → ผังคลี่ดี

**สำคัญ — คำเตือนของ沈孝瞻**:《子平真詮·論十干合而不合》[verbatim] 「今人不知命理，動以本身之合，妄論得失。更有可笑者，書云『合官非爲貴取』，本是至論。而或以本身之合爲合，甚或以他支之合爲合，如辰與酉合，卯與戌合之類，皆作合官，一謬至此，子平之傳掃地矣」 — 沈เตือนว่า"การ合"ต้องเป็นการรวมที่"พา用/忌ออกไปจริง" ไม่ใช่ทุก合ใน四柱จะเป็น "合去"

**Engine rule**: "合去" จะนับเฉพาะเมื่อ:
- ก้านที่มา合 อยู่ "ติด" ก้านที่ถูก合
- ก้านที่ถูก合 เป็น天干"ภายนอก" (年/月/時) ไม่ใช่日干 (สำคัญ — 日干ของตัวเองรวมกับอะไรไม่นับเป็น合去, ตรงตามที่ 沈ยืนยัน)
- ก้านที่มา合 แข็งพอที่จะ "ดึง"

### A2.4.3 合官留殺 / 合殺留官 — สูตรเฉพาะ

ที่มา:《滴天髓·官殺》原注 [verbatim] 「藏官露殺，干神助殺，合官留殺，皆成殺氣，勿使官混也；藏殺露官，干神助官，合殺留官，皆從官象，不可使殺混也」

**สถานการณ์**:
- ผังมี官+殺ทั้งคู่ (官殺混雜) → ทั่วไปถือว่าผังขุ่น
- ถ้ามีก้านที่มา合官ไป (เช่น 甲ถูก己合ไป) → เหลือแต่殺 → ขุ่นกลายเป็นใส (**合官留殺** — ใช้殺เป็นแกน)
- ถ้ามีก้านที่มา合殺ไป (เช่น 庚ถูก乙合ไป) → เหลือแต่官 → ขุ่นกลายเป็นใส (**合殺留官** — ใช้官เป็นแกน)

ตัวอย่าง [paraphrase อิง《滴天髓闡微·官殺》任氏舉例] 「甲己相合，合官留殺故而不以官殺混雜論」

**ผลต่อการอ่านจริง**: นี่คือ "合 ที่ดีมาก" — engine ต้องตรวจหาก่อนสรุปว่าผังขุ่น. ถ้าเจอ 合官留殺/合殺留官 ใส่ flag `CHART_PURIFIED_BY_HE` แล้วเปลี่ยน格ตามที่เหลือ

### A2.4.4 合財/合印/合食 (อื่นๆ)

- **合財**: ถ้า財เป็น用神 → 合住財 = พิการ; ถ้า財เป็น忌神 → 合去財 = ดี
- **合印**: เหมือนกัน (ใช้ตัวเข้ามา合)
- **合食**: เหมือนกัน

**กฎสำคัญ**: ห้าม engine สับสน "合來" (合มาช่วย) กับ "合去" (合ไปเอาออก) — สอง phenomena ตรงข้ามกันโดยสิ้นเชิง

### A2.4.5 大運/流年 合/ปลด ก้านในผังเดิม

- **大運合 用神**: ทำให้用神พิการตลอด10ปี → reading ทศวรรษนั้นไม่ดี
- **大運合 忌神**: ทำให้忌神พิการตลอด10ปี → reading ทศวรรษนั้นดี
- **流年合**: เฉพาะปี
- **流年เปิด合เดิม (สวมก้านที่剋ก้าน合)**: ปลด用神ที่ถูก合住ในผังกำเนิด → reading ปีนั้นดีขึ้น (ปลดพันธ)
- **流年เปิด合ที่合忌**: เปิดให้忌神กลับมาทำงาน → reading ปีนั้นแย่

**Engine rule**:
- เก็บ"ก้านพันธ"ระดับผังกำเนิดเป็น baseline
- คำนวณ"การเพิ่ม-ลดพันธ"จากแต่ละ大運/流年
- map กลับเข้า 用神/喜忌 ของผัง

## A2.5 ข้อสำนักขัดแย้ง + default ที่เลือก (school conflicts + chosen defaults)

> ส่วนนี้สรุปทุกจุดที่ 沈孝瞻 / 徐樂吾 / 任鐵樵 / 萬民英(三命通會) / 徐大升(淵海子平) ขัดแย้งกัน พร้อม default ที่ engine เลือก

### ข้อขัดแย้งที่ 1 — 貪合忘冲 เป็นสัมบูรณ์หรือมีเงื่อนไข?

| สำนัก | จุดยืน |
|---|---|
| 沈孝瞻《子平真詮》 | สัมบูรณ์: "三會六合，可以解之" |
| 任鐵樵《滴天髓》 | มีเงื่อนไข: 合ที่มาช่วยต้องเป็น喜神 ถ้าเป็น忌神 ก็ยัง"ไม่ดี" — [verbatim] 「若忌合而合之，比沖愈凶也」 |
| 徐樂吾《評註》 | มีเงื่อนไข: "看地位與性質之如何而定" |

**Default ที่ engine เลือก**: **มีเงื่อนไข** — ใช้ของ任 + 徐 เพราะมีรายละเอียดสูงกว่า. เงื่อนไข: ก้าน/กิ่งที่มา合ต้องอยู่ติด AND ไม่ใช่忌神-ที่-แรงกว่า-用神-มาก

### ข้อขัดแย้งที่ 2 — 隔位 合 ทำงานได้ไหม?

| สำนัก | จุดยืน |
|---|---|
| 沈孝瞻 | ทำงานเป็น "半合" — "得十之二三" (ออกฤทธิ์เพียง 2–3 ใน 10 ส่วน) |
| 徐樂吾 | แล้วแต่กรณี — บางครั้งเต็ม บางครั้งไม่ |
| 任鐵樵 | ผ่านตัวอย่างยอม 遙合 บ่อย, ถ้า化神有勢 |

**Default ที่ engine เลือก**: **ลดแรงตามระยะทาง** (ไม่ตัด) — เก็บ flag `DIST_FAR` ภายในระบบ; แต่ถ้ามีก้านขวาง"剋" → ตัดทิ้งตามที่ 沈ยืนยันชัด

### ข้อขัดแย้งที่ 3 — 六合 ละลาย 冲 ได้ไหม?

| สำนัก | จุดยืน |
|---|---|
| 沈孝瞻 | ได้ — ตรง |
| 任鐵樵 | ได้ แต่อ่อนกว่า 三合 มาแก้ — [verbatim] 「冲得合而靜者則易」 |
| 萬民英《三命通會》 | ได้ — ตามตำราดั้งเดิม |
| 盲派 (สมัยใหม่) | บางสายอ้างว่า 六合 แก้冲ไม่ได้ เพราะพันธอ่อน |

**Default ที่ engine เลือก**: **ได้** (ใช้คลาสสิก) — แต่แรงน้อยกว่า 三合 มาแก้; ถ้ามีทั้ง 六合 + 三合 มาแก้พร้อมกัน ให้ใช้ 三合 เป็นหลัก

### ข้อขัดแย้งที่ 4 — 三會 vs 三合 แรงกว่ากัน?

| สำนัก | จุดยืน |
|---|---|
| 沈孝瞻 | ใช้คำ "三會" หมายถึง 三合 (สับสนคำ) |
| 任鐵樵 | 三會 (ทิศเต็มฤดู) แรงกว่า 三合 |
| 三命通會 | ใช้ทั้งสองคำชัดเจน |
| สนใช้สมัยใหม่ | 三會方 > 三合局 |

**Default ที่ engine เลือก**: **三會方 > 三合局** (สมัยใหม่) — แต่ flag warning ถ้าผังมี三會: บางครั้งหมายถึง專旺/方局格 → ไป A1

### ข้อขัดแย้งที่ 5 — 刑/害/破 ใช้ไหม?

| สำนัก | จุดยืน |
|---|---|
| 沈孝瞻 | 三刑 闕疑 (ไม่แน่ใจ) แต่ไม่ตัด |
| 任鐵樵 | ตัด破ทิ้ง [verbatim] 「削之可也」; 刑/害 "俗謬" — ไม่ใช้เลย |
| 萬民英《三命通會》 | ใช้ครบ มีบทเฉพาะ「論六害」「論三刑」「論衝擊」 |
| 徐大升《淵海子平》 | ใช้ครบ มีบท「論十二支相穿」「論十二支相刑」 |

**Default ที่ engine เลือก**: **ใช้ 刑 (low weight); ใช้ 害 (very low); ไม่ใช้ 破** — สังเคราะห์: ตัดตามที่ 任รุนแรงที่สุด แต่ไม่ตัดสุดทาง เพราะ 三命通會 และ 淵海子平 ใช้

### ข้อขัดแย้งที่ 6 — 合而不化 ยัง "ใช้" function ของก้านได้ไหม?

| สำนัก | จุดยืน |
|---|---|
| 沈孝瞻 | "有若無也" — เหมือนไม่มี (พิการสนิท) |
| 任鐵樵 | "羈絆" — ยังอยู่แต่ทำงานไม่เต็มที่ |
| 徐樂吾 | ยังคำนวณเทพได้ แต่ลดน้ำหนัก |
| 盲派 | 大運合八字為合絆, 流年合八字為合動 — แยกชัด |

**Default ที่ engine เลือก**: **พิการชั่วคราว** (ตามทิศ 任) — ก้านยังอยู่แต่ flag `BOUND_INACTIVE` เมื่อเป็น用神. ปลดเมื่อมี流年/大運มาตีก้านที่合

### ข้อขัดแย้งที่ 7 — 反吟 / 伏吟 ปฏิบัติอย่างไร?

| สำนัก | จุดยืน |
|---|---|
| 萬民英《三命通會》 | "二者不利六親，非橫破財，不為吉兆" — มัก凶 |
| 命理正宗 (張楠) | [paraphrase] 「反吟伏吟泪淋淋，不傷自己損他人」 |
| 任鐵樵《滴天髓》 | ไม่กล่าวเฉพาะ — ผ่านการคำนวณ生剋ตามปกติ |
| สมัยใหม่ (ปฏิบัติ) | 反吟 > 伏吟 ในแง่ damage; ทั้งคู่ flag เป็น "กรอบเสา" |

**Default ที่ engine เลือก**: **flag เป็นชั้นเสริม pillar-level event** — ไม่แทนที่การคำนวณ生剋ระดับธาตุ. ใช้ ตรวจระดับเสาก่อน → ถ้าเจอ反吟 → ใส่ pillar-event flag; แล้วยังคงผ่าน decision tree ปกติ

### ข้อขัดแย้งที่ 8 — 月令 辰戌丑未 ควร冲ไหม? (墓庫刑沖之說)

| สำนัก | จุดยืน |
|---|---|
| 俗書 / นิยมโบราณ | "辰戌丑未，最喜刑沖，財官入庫不冲不發" — ต้อง冲เปิด |
| 沈孝瞻《子平真詮·論墓庫刑沖之說》 | [verbatim] 「此說雖俗書盛稱之，然子平先生造命，無是說也。夫雜氣透干會支，岂不甚美？又何劳刑沖乎？」 — ปฏิเสธ |
| 任鐵樵 | [verbatim] 「所謂暮庫逢沖則發者，後人之謬也」 — ปฏิเสธ |

**Default ที่ engine เลือก**: **不冲不发 = ตำราเท็จ** — ไม่ใช้. ใช้แบบ 沈+任: ดู 透干+會支ก่อน, 冲庫 เป็นเรื่องร้าย (สลายราก) ถ้าก้านเป็น用神

## Engineer summary — ลำดับการใช้ A2.0–A2.5 ในจริง

ขั้นตอนที่ engine ต้องเดิน (สั้น):

1. **Parse**: อ่านเสา 4 เสา + 大運/流年 ที่กำลังตรวจ
2. **A2.0**: ระบุปฏิสัมพันธ์ทุกชนิดที่มีในผัง (天干合, 地支會/合/沖/刑/害)
3. **A2.1**: ตรวจ天干合แต่ละคู่ — ตัดสิน 真化/假化/合絆 (ไป A1 ถ้า 真化)
4. **A2.2**: ตรวจ地支 — ใช้ Decision Tree A2.2.1 เป็น default
5. **A2.2.2**: ใช้ Conditional Layer ปรับลำดับ (พลิก default ตามตำแหน่ง/ระยะ/จำนวน/原局-動)
6. **A2.3**: resolve 沖中逢合 / 合處逢沖 / 三合被沖 / 反吟-伏吟 ระดับเสา
7. **A2.4**: map ก้าน/กิ่งที่ถูก合住 หรือ合去 ไปยัง 用神/喜忌 — flag ที่กระทบ
8. **A2.5**: ถ้ามีกรณีที่ขัดแย้งระหว่างสำนัก — ใช้ default ที่กำหนด
9. **Output**: รายการ flag ส่งต่อ A3 (interpretation engine) ที่จะแปลงเป็นคำอ่าน — **ห้ามมีตัวเลข/เปอร์เซ็นต์ในข้อความที่ลูกค้าเห็น** (NO_PERCENT rule)
