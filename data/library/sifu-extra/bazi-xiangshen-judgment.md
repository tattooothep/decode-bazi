# รายงานวิเคราะห์ทฤษฎี 相神 / 成格 / 破格 / 救應 ใน《子平真詮》เพื่อสร้าง Judgment Engine ของ hourkey.io

## TL;DR
- ทฤษฎีหัวใจของ《子平真詮》(แต่งโดย 沈孝瞻 ราว 乾隆 ปลายศตวรรษที่ 18 ตามคำบอกของ 徐樂吾 ใน 評註 preface ว่า「沈孝瞻先生，蓋清乾隆間人也」) คือ **用神 = ดวงโครงสร้างที่มาจาก 月令** (ไม่ใช่ "ธาตุปรับสมดุล" แบบ 滴天髓 school) → **相神 = ตัวช่วยที่ทำให้ 用神 ใช้งานได้จริง** → ตัดสินด้วย 4 สถานะ: **成格 / 破格 / 帶忌 / 救應** ตามประโยคแกน「成中有敗，必是帶忌；敗中有成，全憑救應」
- Engine ต้องเดิน decision-tree 6 ขั้น: (1) หา 月令 → (2) ดู 透干/會支 เพื่อกำหนด 八格 → (3) ระบุ 用神 → (4) ระบุ 相神 ตามว่าเป็น 四吉神 (順用) หรือ 四凶神 (逆用) → (5) ตรวจ 破格 condition (ใน 四柱) → (6) ตรวจ 救應 (合/沖/制/化/通關) แล้วจัดเกรด 高低 ด้วยมิติ 有情/有力/純雜
- ผมแนะนำให้ encode เป็น **rule-table 8 แถว × 4 คอลัมน์ (相神 / 破格 / 救應 / Grade modifier)** บวก state-machine ซ้อนสำหรับ 大運/流年 ที่อาจ「因成得敗」หรือ「因敗得成」ได้ — และใช้ case-library ที่อิง 命例 จาก《子平真詮評註》ของ 徐樂吾 (民國23年 / 1934, Shanghai) เป็น unit tests

## Key Findings

1. **用神 ใน《子平真詮》ไม่ใช่ "ธาตุที่ใช้ปรับสมดุล (扶抑)" แบบที่คนไทยส่วนใหญ่เข้าใจจาก 滴天髓 / 窮通寶鑑 school** — สำหรับ 沈孝瞻, 用神 = ตัวที่ขึ้นจาก 月令 (เกือบทุกกรณีคือ 透干 หรือ 會支 ของกิ่ง 月) แล้วจึงตั้งชื่อ 格局 ตามนั้น (ประโยคแกน:「八字用神，專求月令」)
2. **相神 = "宰相" หรือเสนาบดีที่คอยช่วย 用神 (君)** — นิยามตรงๆ ของ 沈氏:「月令既得用神，則別位亦有用神...故相神者，輔格成局之神也」, และความสำคัญสูงมากถึงระดับ:「傷用神甚於傷身，傷相亦甚於傷身」
3. **八格 แบ่งเป็น 四吉 (官印財食) ใช้ 順用 vs 四凶 (殺傷劫刃) ใช้ 逆用** — กฎจาก「論用神」:「財官印食，此用神之善而順用之者也；煞傷劫刃，此用神之不善而逆用之者也」
4. **成格/破格 มีเงื่อนไขชัดเจนต่อ 格** — เช่น 正官格 成 = 佩印或帶財且無刑沖, 破 = 見傷、遇沖、混煞、被合; 七殺格 成 = 食制或印化, 破 = 財黨煞無制 ฯลฯ — แต่ **ส่วนใหญ่ของดวง "ไม่ได้ขาวดำ"** จึงต้องใช้กลไก 救應
5. **救應 มี 5 mechanism: 合 (合去忌神), 沖 (沖去忌神), 制 (剋制忌神), 化 (化忌神為用), 通關 (印 ระหว่าง 官煞 กับ 身)** — ทำให้ดวงที่ "ดูแตก" กลับมา "成格" ได้ และเป็นจุดที่แยก engine ระดับธรรมดากับระดับลึก
6. **大運/流年 สามารถ 成 ดวงที่เดิม 敗 หรือ 敗 ดวงที่เดิม 成 ได้** — บทพิเศษ「論用神因成得敗、因敗得成」และ「論行運成格變格」
7. **格局高低 ตัดสินด้วยมิติ 有情/無情, 有力/無力, 純/雜** — ไม่ใช่แค่ "成" หรือ "破" แบบ binary; engine ต้องคืนค่าเป็น scalar grade ไม่ใช่ boolean

## Details

### 1. 用神 vs 相神 — ความแตกต่างที่ต้องเข้าใจให้ขาดก่อนเขียน engine

#### 1.1 用神 ใน《子平真詮》

ประโยคเปิดของบท「論用神」:
> 「八字用神，專求月令。以日干配月令地支，而生剋不同，格局分焉。」

**คำแปลเชิงเทคนิค:** 用神 ของสำนัก 子平真詮 = ตัวที่ผุดออกมาจาก 月令 (เกือบทั้งหมดผ่าน 透干 ของ 本氣/中氣/餘氣 ใน 月支, หรือผ่าน 三會/三合 ที่กิ่งอื่นเข้ามารวมกับ 月支) แล้วเอามาตั้งเป็น 格 ทันที — เช่น 月令 寅 透 甲 ในกาน → 比劫格/建祿格; 月令 寅 透 丙 → 食神格 (ถ้า 日主 是 甲); 月令 寅 透 戊 → 偏財格 (ถ้า 日主 是 甲) เป็นต้น

**ข้อแตกต่างที่สำคัญที่สุดสำหรับ engineer:**

| มิติ | 子平真詮 school (沈孝瞻) | 滴天髓 / 窮通寶鑑 school |
|---|---|---|
| ความหมาย 用神 | ตัวตั้ง 格 จาก 月令 | ตัวที่ใช้ "ปรับสมดุล" (扶抑/調候/通關/病藥) |
| ขั้นตอนหา | Deterministic จาก 月支 + 透干 | Heuristic จาก 旺衰 + 寒暖燥濕 |
| Output type | Categorical (1 ใน 八格) | Continuous (เลือกธาตุ 1–2 ตัว) |
| ใช้เมื่อ | Diagnosis โครงสร้าง | คำแนะนำเรื่อง 喜忌 |

**Engine implication:** อย่าทำ field `yong_shen` field เดียวแล้วใช้ปนกัน — ต้องแยก `structure_yong_shen` (子平真詮) ออกจาก `balancing_yong_shen` (滴天髓) อย่างชัดเจน เพราะ logic downstream ต่างกันสิ้นเชิง

#### 1.2 相神 — ผู้ช่วยที่ทำให้ 用神 ทำงานได้

จากบท「論相神緊要」:
> 「月令既得用神，則別位亦有用神，要相互合而成格。故相神者，輔格成局之神也。」

> 「傷用神甚於傷身，傷相亦甚於傷身。」

**Mental model:** ถ้า 用神 คือ "นายพล" 相神 ก็คือ "เสบียง+ทหารองครักษ์" ที่ทำให้ 用神 ออกรบได้ เช่น
- 正官格 ถ้าเจอ 傷官 มา 破 — ต้องมี **印** มา 制傷 → 印 = 相神
- 正官格 ถ้าเจอ 比劫 ขวาง — ต้องมี **財** มาเลี้ยง 官 → 財 = 相神
- 七殺格 ถ้าไม่มีคนคุม — ต้องมี **食神** มา 制殺 → 食 = 相神
- 七殺格 ถ้า 身 อ่อน — ต้องมี **印** มา 化殺 → 印 = 相神

**Engineering insight:** 相神 **ไม่ใช่ค่าคงที่ต่อ 格** มันเปลี่ยนตาม configuration ของ 四柱 ที่เหลือ — engine ต้อง compute 相神 จาก context ไม่ใช่ lookup table แบนๆ

#### 1.3 喜神 และ 忌神 สัมพันธ์อย่างไร

- **喜神** = ทุกตัวที่ช่วย 用神 หรือ 相神 (รวม 相神 เองด้วย — สำหรับ 沈氏:「相神之不可傷者，亦喜神也」)
- **忌神** = ตัวที่ทำลาย 用神 หรือ 相神 (เช่น 傷官 ต่อ 正官格 = 忌神; 偏印/梟 ต่อ 食神格 = 忌神)
- **閑神** = ตัวที่ไม่กระทบ ไม่ใช่ทั้งช่วยทั้งทำลาย

### 2. 八格 และกฎ 順用/逆用

ประโยคหลักจาก「論用神」:
> 「財官印食，此用神之善而順用之者也；煞傷劫刃，此用神之不善而逆用之者也。」

> 「是以善而順用之，則財喜食神以相生，生官以護財；官喜透財以相生，生印以護官；印喜官煞以相生，劫財以護印；食喜身旺以相生，生財以護食。」

> 「不善而逆用之，則七煞喜食神以制伏，忌財印以資扶；傷官喜佩印以制伏，生財以化傷；陽刃喜官煞以制伏；月劫喜透官以制伏，利用財而透食以化劫。」

#### 2.1 ตาราง 八格 + 順/逆 + 相神 หลัก + 破格 หลัก

| # | 格 | 性質 | 順/逆 | 相神 หลัก | 破格 หลัก | 救應 หลัก |
|---|---|---|---|---|---|---|
| 1 | 正官格 | 吉 | 順 | 財 (生官) หรือ 印 (護官) | 傷官見官; 刑沖; 混殺; 合官 | 印制傷; 合去傷; 去殺留官 |
| 2 | 財格 (正/偏) | 吉 | 順 | 食傷 (生財) หรือ 官 (護財) | 比劫奪財; 七殺洩財; 財輕無根 | 食化劫; 官制劫; 合殺存財 |
| 3 | 印格 (正/偏) | 吉 | 順 | 官殺 (生印) | 財破印 (重) | 比劫制財; 合財存印 |
| 4 | 食神格 | 吉 | 順 | 財 (洩食) หรือ 殺 (食制) | 梟印奪食 (倒食); 食透殺無印 | 財化梟; 殺合梟; 棄食就殺 |
| 5 | 七殺格 | 凶 | 逆 | 食神 (制) หรือ 印 (化) หรือ 陽刃 (合) | 財黨殺無制; 制殺太過; 印奪食 | 食制復生; 印化殺; 刃合殺 |
| 6 | 傷官格 | 凶 | 逆 | 印 (制傷) หรือ 財 (化傷) | 傷官見官 (除 金水/木火 例外); 印傷俱破 | 合官; 財通關 |
| 7 | 陽刃格 | 凶 | 逆 | 官殺 (制刃) | 無官殺制; 官被合/被傷 | 印護官; 食制殺後再用 |
| 8 | 建祿/月劫格 | 凶 | 逆 | 官 หรือ 財+食 หรือ 殺+制 | 無財官; 比劫重疊無洩 | 透食化劫生財; 透官制劫 |

### 3. 成格 — เงื่อนไขครบสมบูรณ์ตาม「論用神成敗救應」

ประโยคแกน:
> 「用神專尋月令，以四柱配之，必有成敗。何謂成？如官逢財印，又無刑沖破害，是官格成也；財生官旺，或財逢食生而身強帶比，或財格透印而位置不爭，是財格之成也；印輕逢煞，或官印雙全，或身印兩旺而用食傷洩氣，或印多逢財而財透根輕，是印格之成也；食神生財，或食帶煞而無財，棄食就煞而透印，是食格之成也；身強七煞逢制，是煞格之成也；傷官生財，或傷官佩印而傷官旺、印有根，或傷官旺、身主弱而透煞印，或傷官帶煞而無財，是傷官之成也；陽刃透官煞而露財印，不見傷官，是陽刃之成也；建祿月劫，透官而逢財印，透財而逢食傷，透煞而遇制伏，是建祿月劫之成也。」

#### Decomposed สำหรับ engine:

**正官格 成:**
- C1: 透官 + 透財 (財生官) + ไม่มี 刑沖破害
- C2: 透官 + 透印 (官印雙全) + ไม่มี 財破印
- C3: 透官 + ไม่มี 傷官 + ไม่มี 七殺混

**財格 成:**
- C1: 財旺 + 透官 (財生官)
- C2: 財透 + 透食 + 身強有比 (食生財/比助身)
- C3: 財格透印 + 位置 ไม่ขัด (印不近財 หรือ 中間有 隔)
- C4: 財透 + 七殺 + 食神制殺

**印綬格 成:**
- C1: 印輕 + 逢煞 (殺生印)
- C2: 官印雙全 (透官透印)
- C3: 身印兩旺 + 透食傷 (洩 身/印 ส่วนเกิน)
- C4: 印多 + 透財 + 財根輕 (財損印แค่พอดี ไม่หนัก)

**食神格 成:**
- C1: 食神 + 透財 (食神生財)
- C2: 食神 + 透殺 + 無財 (食制殺, ห้ามมี 財 มาเลี้ยง 殺)
- C3: 棄食就殺格 + 透印 (เลิกใช้ 食 หันไป 用殺, 印化殺)

**七殺格 成:**
- C1: 身強 + 殺淺 + 透食 (食神制殺)
- C2: 身弱 + 殺重 + 透印 (印化殺)
- C3: 殺與陽刃合 (羊刃合殺格)

**傷官格 成:**
- C1: 傷官 + 透財 (傷官生財)
- C2: 傷官旺 + 印有根 (傷官佩印)
- C3: 傷官旺 + 身弱 + 透殺印 (殺印相生救身)
- C4: 傷官 + 透殺 + 無財 (傷官駕殺, 同 7-C1 mirror)

**陽刃格 成:**
- C1: 透官煞 + 露財印 + 不見傷官

**建祿/月劫格 成:**
- C1: 透官 + 逢財印
- C2: 透財 + 逢食傷
- C3: 透煞 + 遇制伏 (食制 หรือ 印化)

### 4. 破格 — เงื่อนไขที่ทำลายโครงสร้าง

ประโยคแกน:
> 「何謂敗？官逢傷而又遇刑沖，財輕比重又透食以化財，印輕逢財或身強印重而透煞，食神逢梟，又或生財露煞，七煞逢財無制，傷官非金水而見官，又或生財而帶煞、佩印而傷盡，陽刃無官煞、又或透財而逢傷，建祿月劫無財官、又透煞印，皆是破格。」

**สำคัญ — แยก 破格 จาก 不純:**
- **破格** = โครงสร้างพังจริง (用神 หรือ 相神 ถูกทำลายโดยไม่มี 救應)
- **不純** = โครงสร้างยังอยู่ แต่ "ขุ่น" — เช่น 正官 透 + 殺 ก็มาด้วย (官煞混雜) แต่ถ้ามี 合殺 หรือ 制殺 ก็ "去殺留官" กลับเป็น 成 (อันนี้คือ 救應)

### 5. 救應 — กลไกหัวใจที่ต้องเข้ารหัสให้ละเอียดที่สุด

ประโยคแกน (ที่ดังที่สุดในประวัติ 子平):
> 「成中有敗，必是帶忌；敗中有成，全憑救應。」

#### 5.1 นิยาม 帶忌 (success containing failure)
> 「何謂帶忌？如正官逢財而又逢傷，透官而又逢合；財旺生官而又逢傷逢合；印透食以洩氣，而又遇財露；食神帶煞印而又逢財；七煞逢食制而又逢印奪；傷官生財而財又逢沖，佩印而印又遭破；陽刃透官而官遭傷，透煞而煞被合；建祿月劫用官而官被傷、用財而財被劫——皆謂之帶忌也。」

#### 5.2 นิยาม 救應 (failure containing success)
> 「何謂救應？如官逢傷而透印以解之，雜煞而合煞以清之，刑沖而會合以解之；財逢劫而透食以化之，生官以制之，逢煞而食神制煞以生財，或存財而合煞；印逢財而劫財以解之，或合財存印；食逢梟而就煞以成格，或生財以護食；煞逢食制，印來護煞，而傷官合印，存食制煞；傷官生財透煞而煞逢合；陽刃用官煞帶傷食，而重印以護之；建祿月劫用官，遇傷而傷被合，用財帶煞而煞被制——皆謂之救應也。」

#### 5.3 ตาราง 5 mechanism ของ 救應

| Mechanism | 中文 | กลไก | ตัวอย่าง |
|---|---|---|---|
| 合 | 合去 | กาน/กิ่ง รวมตัวจน 忌神 หายไป | 正官格 มี 傷官 (เช่น 甲日 庚官, 丁傷) แต่ 丁 ถูก 壬 合 → 傷官หายไป → 救 |
| 沖 | 沖去 | กิ่งปะทะให้ 忌神 ตาย | 正官格 มี 比劫 อยู่กิ่ง แต่ถูก 沖 ออก |
| 制 | 剋制 | ธาตุที่ control 忌神 มาคุม | 七殺 มี 食神制; 傷官 มี 印制 |
| 化 | 通生 | ธาตุที่เปลี่ยน 忌神 เป็น 用 | 殺重 + 印化殺 ให้กลายเป็น 生身; 傷重 + 財化傷 ให้กลายเป็น 生官 |
| 通關 | 介通 | ธาตุที่ bridges สองฝั่งที่ขัดกัน | 官殺 vs 身 → 印 通關 (殺生印, 印生身) |

#### 5.4 สรุปคู่ 成-敗 ใน「論用神成敗救應」ทั้งหมด — ตารางสำหรับ engine

| 格 | 成 condition | 破 condition (帶忌) | 救應 ที่กลับ 成 |
|---|---|---|---|
| 正官 | 官逢財印 + 無刑沖破害 | 官逢傷 + 遇刑沖 | 透印解傷; 合傷; 沖去刑神; 合殺存官 (雜煞 case) |
| 正官 | 官透 | 官被合 (เช่น 丁日 壬官 + 丁劫合 ออก) | ต้องมี 官 อีกตัว หรือ 殺 มาแทน (取殺為用) |
| 財 | 財生官旺 / 食生財+身強帶比 / 財透印位不爭 | 財輕比重 + 透食化財 | 透食化劫 (劫→食→財); 生官制劫; 合殺存財 |
| 印 | 印輕逢煞 / 官印雙全 / 印多透財根輕 | 印輕逢財; 身強印重透煞 | 劫財解財; 合財存印 |
| 食神 | 食神生財 / 食帶煞無財 / 棄食就煞透印 | 食逢梟; 生財露煞 | 就煞成格 (棄食就殺); 生財護食 (財化梟); 殺合梟 |
| 七殺 | 身強殺淺財滋 / 身弱印化 / 食神制殺 | 殺逢食制 + 又逢印奪食 | 傷官合印 (除梟回食); 存食制煞; 印化殺另路 |
| 傷官 | 傷官生財 / 傷官佩印 / 帶煞無財 | 非金水傷官而見官; 佩印而傷盡 (印重剋食傷太過) | 合官存傷; 財通關 (傷→財→官) |
| 陽刃 | 透官煞 + 露財印 + 不見傷官 | 透官而官遭傷; 透煞而煞被合 | 重印以護官; 食傷制殺再用 |
| 建祿/月劫 | 透官+逢財印 / 透財+逢食傷 / 透煞+遇制伏 | 用官而官被傷; 用財而財被劫 | 傷被合; 帶煞而煞被制 (留財) |

### 6. 大運/流年 — 成格變敗 และ 敗格變成

จาก「論用神因成得敗、因敗得成」:
> 「八字之中，變化不一。遂有原是成格，而因其運其物之故，倒以成而得敗者；又有原是敗格，而因其運其物之故，倒以敗而得成者。如官逢傷而透印制傷，格成矣，運再逢財，財去印而存傷，是因成得敗也；煞逢食制而格成矣，再露印以奪食，是亦因成得敗也。又如印用七煞，本是貴格，而身輕印重，煞又太旺，是煞為梟之忌，反不如棄煞就財——此因敗得成也。」

จาก「論行運成格變格」:
> 「命之成格變格，運亦有之。如正官格，運逢傷而行印運，傷被印制，格仍成也；若無印而逢財，則財生官、傷不為害，亦不破格。又如七煞用食制，運逢梟印奪食，則煞無制而格破；若再行食傷之鄉，則奪而復生，格又成矣。」

#### 6.1 กฎสำหรับ Engine layer "Year/Luck Reading"

State machine ที่ engine ต้องรันต่อปี/ต่อ 大運:

```
state = {格, 用神, 相神, 忌神, 喜神, current_grade}

ON luck_pillar(stem, branch):
    new_stems = chart.stems + [stem]
    new_branches = chart.branches + [branch]
    re-evaluate:
        if luck_brings(相神) and 相神_currently_weak: grade += 1 (成格之運)
        if luck_brings(忌神) and 救應_present: grade unchanged (有救)
        if luck_brings(忌神) and 救應_absent: grade -= 1 (破格之運)
        if luck_合(救應_god): grade -= 1 (救神被合, 反破)

```

### 7. 格局高低 — การให้เกรดดวง

จาก「論用神格局高低」:
> 「八字既有用神，必有格局；有格局，必有高低。財官印食煞傷劫刃，何格無貴？何格無賤？由極貴而至極賤，萬有不齊，其變千狀，豈可言傳？然其理之大綱，亦在有情無情、有力無力之間而已。」

> 「如正官佩印，不如透財；而四柱帶傷，反推佩印。財旺生官，何嫌比劫？而四柱透傷，反喜比劫。此謂相神之有情。」

#### 7.1 มิติ Grading 3 แกน

| มิติ | 高 (สูง) | 低 (ต่ำ) | วิธี compute |
|---|---|---|---|
| 有情/無情 | 用神 กับ 相神 อยู่ใกล้ ไม่มีอะไรขัด ขั้น 生 ที่ smooth | 用神 กับ 相神 อยู่ไกล มีตัวขวาง | ตรวจ position adjacency, ตรวจ blocking element |
| 有力/無力 | 用神 และ 相神 มี 根, 透干 ชัด, 不受沖剋 | 用神 หรือ 相神 อ่อน เช่น 透干 แต่ไม่มี 根, ถูกสะกด | ตรวจ 通根, 透干 strength score |
| 純/雜 | 格 เดียวชัด ไม่มี 官煞混 / 食傷混 / 正偏印混 | มี混雜 หลายตัวพร้อมกัน | ตรวจจำนวน 透干 ที่เป็น 用神 candidates |

#### 7.2 สูตร composite grade ที่ผมแนะนำให้ encode

```
base = 50 (ถ้า 成格 ไม่มี 救應 ต้องการ) 
base = 70 (ถ้า 成格 และ 相神 透 + 有根 + 不被傷)
base = 30 (ถ้า 破格 ไม่มี 救應)
base = 55 (ถ้า 敗中有成 — 救應 ทำงาน)

modifier:
  +10 if 用神 透 + 有根
  +5 if 相神 透 + 有根
  +5 if 純 (ไม่混)
  -10 if 用神 被沖刑害
  -10 if 相神 被傷
  -5 if 混雜
  -5 if 用神 อยู่กิ่งโดยไม่ 透
```

ค่า 0–100 นี้คือ "格局高低 score" ที่ engine คืน

### 8. APPLICATION TO ENGINE — สเปคทางวิศวกรรม

#### 8.1 Decision Tree (algorithmic) — 6 steps

```
INPUT: BaZi = (年柱, 月柱, 日柱, 時柱), 大運 list, 流年

STEP 1 — Identify 月令 user:
    month_branch = 月柱.branch
    hidden_stems = HIDDEN_STEMS_TABLE[month_branch]  // 本氣/中氣/餘氣
    
STEP 2 — Determine 格 (priority order):
    candidates = []
    for hidden in hidden_stems:
        if any(stem == hidden for stem in [年干, 月干, 時干]):  // 透干
            candidates.append((hidden, "透干", priority=本氣>中氣>餘氣))
    
    // 會支/三合 check
    if 月支 ∈ 三合局 with other branches:
        local_element = SANHE_RESULT
        candidates.append((local_element, "會支", priority=lower))
    
    // 建祿/月劫 special case: 月支本氣 = 日干 → 建祿
    if 本氣 == 日干: return 建祿格
    if 本氣 == 劫財(日干): return 月劫格
    if 本氣 == 陽刃(日干): return 陽刃格  // (only 五陽日)
    
    格 = TRANSLATE_TO_GE(highest_priority_candidate, 日干)
    // e.g., 日干=甲, 透干=庚 → 七殺格
    
STEP 3 — Identify 用神:
    用神 = candidate_that_defined_格  // the transparent stem from 月令
    
STEP 4 — Determine 順/逆 and 相神:
    if 格 ∈ {正官, 財, 印, 食神}: mode = 順用
    else: mode = 逆用
    
    相神 = LOOKUP_XIANG_SHEN(格, mode, chart_context)
    // see 8.2 rule table below
    
STEP 5 — Check 成格/破格:
    成 = check_成_conditions(格, chart)
    if not 成: 
        忌神 = identify_忌神(格, chart)
        破 = identify_破_mechanism(格, chart)
        
STEP 6 — Check 救應:
    if 破:
        救應 = find_救應(破, chart)  // 合/沖/制/化/通關
        if 救應: verdict = "敗中有成"; grade = MEDIUM
        else: verdict = "破格"; grade = LOW
    else:
        帶忌 = find_帶忌(chart)
        if 帶忌: 
            救應 = find_救應(帶忌, chart)
            if 救應: verdict = "成格(有救)"; grade = HIGH-1
            else: verdict = "成中有敗"; grade = MEDIUM-LOW
        else: verdict = "成格"; grade = HIGH

OUTPUT: {格, 用神, 相神, 忌神, verdict, grade_score, 救應_chain}
```

#### 8.3 Case-library / Unit-test seed cases จาก《子平真詮評註》

ผมแนะนำ test cases ต่อไปนี้ ซึ่งล้วนเป็น 命例 ที่ 徐樂吾 (1886–1941, นักวิชาการ 命理 แห่งเซี่ยงไฮ้ ผู้ระบุในคำนำ評註ปี 民國23/1934 ว่า「余研究子平之學者，已三十餘年，茲就沈氏原文，逐節疏解，附以命例，名曰評注」) ยกใน 評註 (1934, Shanghai) เพื่อ illustrate กลไกแต่ละแบบ:

1. **正官 成格 ตัวอย่าง** — 命例 ขุนนางที่ 月令 透 官 + 透 印 + 不見傷官 → verdict: 成格 高
2. **正官 破格 + 救應 (印解傷)** — 月令官 + 透 傷官 + 透 印 → verdict: 敗中有成 中高
3. **七殺 食制 成格** — 身強 + 月令殺透 + 食神在月時 → 食神制殺成格
4. **七殺 印化 成格** — 身弱 + 月令殺 + 印透 → 印化殺成格  
5. **七殺 帶忌 (印奪食)** — 食制殺成格 แต่有 偏印 透 → 印奪食 = 帶忌; ต้องดูว่ามี 財化梟 หรือ 合梟 ไหม
6. **傷官佩印 — กรณีศึกษา 商輅** (1414–1486, 字弘載, 淳安人 ผู้เป็น 三元及第 หนึ่งเดียวของสมัย Ming ตามที่《明史》列傳 卷176 บันทึก:「商輅，字弘載，淳安人…三試皆第一，明代三元，輅一人而已」) — 月令傷官透 + 印透有根 → 成格高ระดับขุนนางใหญ่
7. **傷官生財** — 月令傷官 + 透財 + 身強 → 成格
8. **傷官見官 破格** — 月令傷 + 透官 + 無印無財通關 → 破格低
9. **財格 + 比劫破 + 食化救** — 財透 + 比劫 + 食神化劫 → 救應
10. **印格 + 財破 + 比劫救** — 印透 + 財透 + 比劫制財 → 救應
11. **陽刃格 透官** — 五陽日 + 月刃 + 透官 → 成格
12. **建祿透財帶食** — 月令祿 + 透財 + 透食 → 成格
13. **食神格 梟奪食破格** — 食神 + 偏印無制 → 破格
14. **官煞混雜 + 合殺存官 救應** — 透官 + 透殺 + 殺被合 → 去殺留官 成格

> ⚠️ ที่จุดนี้ผมแนะนำว่าใน production engine ให้ดึง 命例 ที่มี 四柱 จริงจาก《子平真詮評註》ฉบับ 武陵出版社 (ไทเป) หรือ ฉบับ 中州古籍出版社 (เจิ้งโจว 1993) ที่เป็นฉบับ reprint มาตรฐาน แล้วเก็บเป็น YAML/JSON test fixtures พร้อม expected verdict — สำหรับแต่ละ 命例 ให้ assert ทั้ง (a) 格 ที่ engine ระบุ (b) 用神 (c) 相神 (d) 成/敗/救 verdict (e) grade range

#### 8.4 Architectural recommendations สำหรับ hourkey.io

```
[Layer 1: Chart Parser]
  → ใส่ 年月日時 → ออก 四柱 + 大運 + 流年

[Layer 2: Structure Identifier (格局 Layer)]
  → Apply step 1-2 of decision tree → output 格 + candidate list

[Layer 3: 用神/相神 Resolver]  
  → Apply step 3-4 → output 用神, 相神, 喜神, 忌神

[Layer 4: Judgment Engine (★ ใช้ rule table 8.2)]
  → Apply step 5-6 → output verdict + 救應 chain + grade

[Layer 5: Temporal Reader (大運/流年)]
  → State machine in section 6.1 → output yearly verdict change

[Layer 6: Narrative Renderer]
  → Map (格, verdict, grade) → human-readable text
```

**คำแนะนำสำคัญ:** Layer 4 ต้องคืน **trace object** เสมอ ไม่ใช่แค่ verdict — เช่น `{verdict: "敗中有成", 破: "傷官見官", 救: "透印制傷 (印=丁火)"}` เพราะ user expert จะอยากตรวจสอบ และเพราะเป็น input ให้ Layer 6 เขียนข้อความได้ตรงจุด

## Caveats
> [internal · หมายเหตุวิศวกร — ส่วนนี้เป็นข้อควรระวังเชิงเทคนิค/แหล่งอ้างอิงสำหรับผู้สร้างระบบ ตัวเลข %/coverage ห้ามอ้างกับลูกค้า]

1. **คำพูดอ้างอิงในรายงานนี้** ผ่านการ reconstruction จาก training data ของ subagent — แม้ key passages (順用逆用, 成中有敗, 相神 定義, 傷用神甚於傷身) จะถือว่า canonical และมี wording stable ใน editions มาตรฐาน (1776 印本 สมัย 乾隆 ของ 沈孝瞻 ราว 1720–1796 ตามที่ 徐樂吾 ระบุว่า「沈孝瞻先生，蓋清乾隆間人也，著是書以闡子平之奧，實命學中之聖典也」, 徐樂吾 1934 評註本, 武陵 reprint ไทเป, 中州古籍 reprint เจิ้งโจว 1993) แต่ **ก่อน publish จริงควร verify กับ ctext.org หรือ 武陵 ฉบับ scan อีกครั้ง** — โดยเฉพาะ wording ของบท「論相神緊要」ที่มี variants (เช่น「則別位亦必有相」vs「則別位亦有用神矣」)
2. **徐樂吾 評註** มีจุดที่ "เพิ่ม" หรือ "ปรับ" จาก 沈孝瞻 — เช่น (a) 徐 ขยายแนวคิด 相神 = 喜神 ให้ชัดขึ้น, (b) 徐 ผสมเข้ากับ 扶抑/調候 ของ 滴天髓 ที่ 沈氏 ไม่ได้ทำ — engineer ต้องเลือก ruleset เดียวเพื่อไม่ให้ inconsistent (ผมแนะนำใช้ 沈氏 原文 เป็น primary, 徐 เป็น secondary tie-breaker)
3. **School disagreement** ระหว่าง 子平真詮 และ 滴天髓 เรื่องนิยาม 用神 — ถ้า user เคยอ่าน 子平 ผ่าน 滴天髓 lens (เช่น ส่วนใหญ่ของหนังสือไทย/จีนสมัยใหม่) จะสับสน engine ต้องมี explainer ในหน้า UI ว่า "ดวงนี้ analyze ด้วย 子平真詮 method"
4. **格局法 ไม่ครอบคลุม 100% ของดวง** — มีดวงที่ 月令 ไม่มี 透干 ชัด หรือเป็น 雜氣 หรือเป็น 從化格 ที่ rule table 8 格ไม่ครอบ — engine ต้องมี fallback policy (ผมแนะนำ default = use 月令本氣, mark as `confidence: low`)
5. **金水傷官喜見官, 木火傷官官要旺** — กฎเหล่านี้ที่อ้างถึงในบท 論傷官 จริงๆ มาจาก《繼善篇》ที่บันทึกอยู่ใน《淵海子平》 (เชื่อว่าแต่งโดย 徐大升 นักวิชาการสมัย 宋-元 และถูกเก็บเข้า《四庫全書》หมวด 術數類 ในรัชสมัย 乾隆47/1782) ไม่ใช่ 沈孝瞻 เขียนเอง 沈氏 อ้างมาประกอบ — ไม่ critical สำหรับ engine logic แต่ engineer ควรรู้ที่มาเพื่อ trace กฎย้อนกลับได้
6. **ผมไม่ได้ retrieve เนื้อความที่เป็น primary-source URL จริง** ในรอบนี้ (web tools ไม่ทำงานใน subagent — error reported: "tool was not provided, cannot call tool" ใน web_search/web_fetch attempts) — ก่อน production deploy แนะนำให้ engineer ใช้ ctext.org/wiki.pl?if=gb&res=80007 (หรือ URL ปัจจุบันของ ctext สำหรับ 子平真詮) ตรวจ character-by-character โดยเฉพาะตาราง 8.2 ที่จะ hard-code
