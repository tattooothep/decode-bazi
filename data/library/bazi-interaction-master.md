## 1) กฎทองสูงสุด

### 1.1 Mechanism ก่อน Favorability

คะแนนหรือแรงของปฏิกิริยาเป็นเพียง **แรงกลไก** เช่น รวมแรง ชนแรง ตรึงแรง เปิดคลังแรง แต่ผลดี/ร้ายจริงต้องดูว่าแรงนั้นกระทบอะไร:

- ส่งเสริม 用神 หรือคุม 忌神 = มีแนวโน้มดี
- บั่นทอน 用神 หรือปล่อย 忌神 = มีแนวโน้มเสีย
- เสริม 忌神 แม้กลไกเป็น 合 หรือ 生 ก็อาจเสีย
- ชง 忌神 ที่ถูกกักไว้ บางครั้งเป็นผลเสียเพราะปล่อยของเสียออกมา

### 1.2 ห้ามตัดสิน 用神 จากดวงดิบทันที

ขั้นที่ถูกต้องคือ:

1. แปลงเสาและก้านซ่อน
2. ประเมินฤดูกาลและราก
3. resolve ปฏิกิริยากิ่งดินและก้านฟ้า
4. คำนวณกำลังธาตุใหม่
5. แล้วจึงตัดสิน Day Master 旺衰, 格局, 用神, 忌神

### 1.3 合 ไม่ได้แปลว่า 化 เสมอ

เมื่อเจอ 合 ให้จำแนกเป็น 3 ระดับ:

- **真化 / true transform**: รวมแล้วแปรธาตุจริง ต้องได้ฤดู มีราก มีตัวนำ และไม่มีตัวขัด
- **合而不化 / bound-no-transform**: รวมแต่ไม่แปร เป็นกรณีที่พบบ่อยที่สุด ทั้งสองฝ่ายถูกตรึง ลดหน้าที่เดิม
- **争合 / 妒合 / contested combine**: มีตัวแย่งรวม การรวมไม่สนิท เกิดความลังเล/แทรกซ้อน

## 3) ลำดับการประมวลผลหลัก

### 3.1 Pipeline สำหรับ engine

- 1. Parse 4 pillars → stems + branches; expand each branch to hidden stems (layer_5).
- 2. Apply layer_8 seasonal multipliers to baseline element strengths from 月令.
- 3. PHASE 0: proximity gating (layer_7 phase_0 + layer_6 distance).
- 4. PHASE 1: resolve 三會/三合 — lock branches.
- 5. PHASE 2: resolve 六合 vs 六冲 with 貪合忘冲 + magnitude/1.2 threshold.
- 6. PHASE 3: resolve 刑/害/破 on surviving branches; apply 合化解刑 shielding.
- 7. PHASE 4: resolve tomb open/close.
- 8. PHASE 5: resolve stem 合/克 using FINAL branch roots; grade 合化 (真化/不化/爭合).
- 9. PHASE 6: recompute element tallies & DM 旺衰; THEN assign 用神/忌神; THEN flip mechanism scores to favorability scores.
- 10. Aggregate to the module's scale (e.g. §7 maps -100..+100 → 0..20 with ×1.3 penalty per 凶力倍於和合).

### 3.3 หลักชนะกันเมื่อปฏิกิริยาซ้อน

- 三会方局 มักแรงที่สุด เพราะเป็นการรวมทิศ/ฤดู
- 三合局 รองลงมา เป็นการรวมกำเนิด-รุ่ง-คลังของธาตุ
- 六合 มีแรงตรึงสูง โดยเฉพาะเมื่ออยู่ใกล้เสาวันหรือเดือน
- 六冲 ทำงานเต็มเมื่อไม่มี 合/局 มาดึงรั้ง และแรงขึ้นมากเมื่อแตะ 月令 หรือ 日支
- 刑/害/破 เป็นแรงเรื้อรัง/ซ่อนเร้น ทำงานหลังจาก 合/冲 ถูก resolve แล้ว
- 墓库 辰戌丑未 ต้องดูเปิด/ปิดคลังหลังปฏิกิริยากิ่งหลัก
- 天干五合/克 ต้องดูหลังสถานะรากของกิ่ง เพราะก้านไม่มีราก = 虚浮

## 11) รูปแบบคำตอบมาตรฐาน

เมื่อผู้ใช้ถามให้วิเคราะห์ดวง ให้ตอบตามโครงนี้:

1. **ข้อมูลที่รับมา**: ระบุสี่เสา/ดวงจร/สิ่งที่ยังขาด
2. **ฐานกำลังดวง**: Day Master, เดือนเกิด, ฤดูกาล, ราก, hidden stems
3. **ปฏิกิริยาที่พบ**: แยก 天干, 地支, 藏干, 墓库, ดวงจร
4. **ลำดับการ resolve**: อธิบายว่าอะไรเกิดก่อน อะไรถูกลดแรง อะไรชนะ
5. **ผลต่อ用神/忌神**: ห้ามใช้คำว่าดี/ร้ายก่อนผ่านชั้นนี้
6. **คำแปลชีวิตจริง**: งาน เงิน คู่ สุขภาพ บ้าน ลูก ตามตำแหน่งเสา
7. **ระดับความมั่นใจ**: สูง/กลาง/ต่ำ พร้อมเหตุผล
8. **ข้อควรระวัง**: ไม่ฟันธงเกินข้อมูล ไม่ใช้แทนแพทย์ กฎหมาย หรือการเงินมืออาชีพ

## 12) ภาษาและน้ำเสียง

- อธิบายให้คนไทยเข้าใจ แต่เก็บคำจีนสำคัญไว้
- **ฟันธงเหตุการณ์/ความสัมพันธ์ได้เด็ดขาดตามน้ำหนักดวง** ไม่ต้องถนอมจนผลจริงเพี้ยน · พูดผลร้ายตรงๆ ได้ ไม่ใช่ทำให้กลัวลอยๆ แต่เป็นการบอกความจริงตามดวง
- **สงวนคำผ่อน "แนวโน้ม / มีเงื่อนไข / ควรระวัง" ไว้เฉพาะ 3 เรื่องเท่านั้น:** (1) สุขภาพ — ห้ามวินิจฉัยโรค (2) อายุขัย — ห้ามทำนายตาย (3) กฎหมาย/การเงินมืออาชีพ · นอกเหนือจาก 3 เรื่องนี้ให้พูดชัดตามน้ำหนักดวง
- ถ้าข้อมูลขัดกัน ให้ระบุว่าสำนักต่างกันตรงไหน และเลือกกฎที่สอดคล้องกับลำดับ engine มากที่สุด
- ถ้าผู้ใช้ต้องการคำตอบสั้น ให้สรุป 3 ชั้น: ปฏิกิริยา → กลไก → ผลเมื่อผ่าน用神/忌神

## 13) ข้อห้ามสำคัญ

- ห้ามบอกว่า 合 = ดีเสมอ
- ห้ามบอกว่า 冲 = ร้ายเสมอ
- ห้ามบอกว่า 合 ทุกคู่แปรธาตุทันที
- ห้ามอ่านคะแนนดิบเป็นดีร้ายโดยไม่ผ่าน context modifier
- ห้ามละเลย 月令, 通根, 藏干
- ห้ามอ่าน 刑/害/破 ก่อน resolve 合/冲
- ห้ามตัดสินดวงจรปีเดียวโดยไม่ดู 大运 ที่เป็นเวทีใหญ่
- ห้ามใช้ปาจื้อแทนคำแนะนำทางแพทย์ กฎหมาย หรือการเงิน
